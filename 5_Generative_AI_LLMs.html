🗣️ Natural Language Processing ke Fundamentals

Text ko machine-readable banane ki journey — raw words se meaning tak

📌NLP Kya Hai?

Natural Language Processing (NLP) AI ki woh branch hai jo computers ko human language — text ya speech — ko samajhne, process karne, aur generate karne ki ability deti hai. Jaise Google Translate, ChatGPT, Siri — sab NLP use karte hain.

1.1 Tokenization

🔤 Tokenization — Text ko Todna

Kya hai Tokenization?

Tokenization ek aisi process hai jisme hum ek bade text string ko chhote meaningful pieces (tokens) mein todte hain. Ye tokens words, subwords, characters, ya phir punctuation marks ho sakte hain.

💡Beginner Intuition

Soch — jaise hum ek sentence ko bol rahe hain "Main school jaata hoon" — toh yeh sentence ke 4 words hain. Ye 4 words hi basically tokens hain! Machine bhi text ko aise hi chhote pieces mein todti hai process karne ke liye.

Tokenization ke Types

TypeKya Karta HaiExampleUse Case
Word TokenizationText ko words par split karta hai"Hello World" → ["Hello", "World"]Basic NLP tasks
Sentence TokenizationText ko sentences mein todta haiPara → ["Sent1.", "Sent2."]Summarization
Subword TokenizationRare words ko subwords mein todta hai"playing" → ["play", "##ing"]BERT, GPT models
Character TokenizationHar character alag token"Hi" → ["H","i"]Language modeling
BPE (Byte Pair Encoding)Common pairs ko merge karta haiGPT-style tokenizationLLMs

BPE (Byte Pair Encoding) — GPT ka Secret

Modern LLMs jaise GPT, Claude, Llama sab BPE tokenization use karte hain. Iska idea simple hai:

1
Start with characters

Pehle har character alag token hota hai: "l", "o", "w" etc.

2
Find common pairs

Sabse zyada baar aane wali character pairs dhundho: "l"+"o" = "lo"

3
Merge them

Pair ko merge karo aur vocabulary mein add karo.

4
Repeat

Desired vocabulary size tak repeat karo (e.g., 50,000 tokens).

Token Visualization — "Unbelievable" ko kaise tokenize karte hain
Un believ able → 3 tokens
I love eating biryani ! → 5 tokens
Python — Tokenization Example 
# Basic Word Tokenization using NLTK
import nltk
from nltk.tokenize import word_tokenize, sent_tokenize

text = "Main ek AI engineer hoon. Mujhe Python bahut pasand hai!"

# Word tokenization
words = word_tokenize(text)
print("Words:", words)
# Output: ['Main', 'ek', 'AI', 'engineer', 'hoon', '.', 'Mujhe', 'Python', ...]

# Sentence tokenization
sents = sent_tokenize(text)
print("Sentences:", sents)
# Output: ['Main ek AI engineer hoon.', 'Mujhe Python bahut pasand hai!']

# ---- GPT-style BPE Tokenization (tiktoken) ----
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4")
text2 = "Artificial Intelligence is amazing!"

tokens = enc.encode(text2)
print("Token IDs:", tokens)
# Output: [9470, 35915, 11, 1002, 374, 8056, 0]

print("Total tokens:", len(tokens))
# Output: 7

# Token wapas text mein convert karo
for token in tokens:
    print(repr(enc.decode([token])), "→ ID:", token)
⚠️Common Mistake

Bahut log sochte hain ki 1 word = 1 token. Yeh GALAT hai! "unbelievable" = 3 tokens ho sakta hai. Iska directly asar padta hai LLM ke cost aur context window par. Always token count check karo, word count nahi!

1.2 Stemming

✂️ Stemming — Root Word Dhundho

Stemming ek aise process hai jisme hum kisi word ke ending (suffix) ko hataate hain taaki us word ka root (stem) mile. Yeh process thodi rough hoti hai — exact root nahi, sirf approximate stem milta hai.

✅ Stemming kab kaam aata hai
  • Search engines mein (fast retrieval)
  • Jab speed important ho accuracy se zyada
  • Information retrieval tasks
  • Simple text classification
❌ Stemming ki limitations
  • "studies" → "studi" (galat English word)
  • "better" → "better" (no change, missed)
  • Context ko samajhta nahi
  • Over-stemming (words galat ban jaate hain)
Python — Stemming 
from nltk.stem import PorterStemmer, SnowballStemmer

porter = PorterStemmer()
snowball = SnowballStemmer("english")

words = ["running", "studies", "happily", "better", "eating"]

print("Word         Porter      Snowball")
print("-"*40)
for w in words:
    print(f"{w:13} {porter.stem(w):12} {snowball.stem(w)}")

# Output:
# Word         Porter      Snowball
# ----------------------------------------
# running      run         run
# studies      studi       studi    ← Thoda rough!
# happily      happili     happili
# better       better      better   ← Kuch nahi hua
# eating       eat         eat

2017 mein Google ke researchers ne ek paper publish kiya: "Attention Is All You Need". Isme ek naya architecture propose kiya gaya — Transformer. Aaj duniya ke sabse powerful AI models (GPT-4, Claude, Gemini) sab Transformer architecture par based hain.

Transformer Architecture Overview
2.1 Attention Mechanism

👁️ Attention Mechanism — Dhyan Kaha Lagayen?

Attention mechanism ka core idea yeh hai: Jab hum ek word ko process karte hain, toh hume sentence ke baaqi words ka kitna importance dena chahiye?

💡Real-World Analogy — Exam Mein Padhna

Socho tum ek question padh rahe ho: "The bank near the river was closed." Yahan "bank" ka matlab hai nadi ka kinara, na ki paisa wali bank. Tumhara brain automatically "river" word ko zyada attention deta hai "bank" ke context ko samajhne ke liye. Yahi kaam Attention Mechanism karta hai!

Attention Ka Mathematical Formula

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V
Jahan Q = Query, K = Key, V = Value, dₖ = Key dimension

Q, K, V Kya Hain? (Intuition)

ComponentAnalogyRole
Query (Q)Librarian se question: "Mujhe AI books chahiye"Current word kya dhundh raha hai
Key (K)Har book ka label: "AI", "Science", "History"Dusre words ki identity/tag
Value (V)Book ka actual contentJo information actually pass hogi
1
Q, K, V matrices compute karo

Input embedding ko 3 alag weight matrices se multiply karo → Q, K, V matrices milti hain.

2
Attention scores compute karo

Q aur K ka dot product lo → similarity scores milte hain. √dₖ se divide karo (scale).

3
Softmax apply karo

Scores ko probabilities mein convert karo (0 to 1) — ye attention weights hain.

4
Value se multiply karo

Attention weights ko V se multiply karo → final context-aware representation milti hai.

2.3 Multi-Head Attention

🎭 Multi-Head Attention — Ek Saath Kaafi Nazariye

Ek single attention head ek hi type ka relationship pakad sakta hai. Lekin language mein bahut tarah ke relationships hote hain — grammatical, semantic, coreference etc. Multi-Head Attention ek baar mein multiple attention heads parallel mein chalata hai.

💡Analogy — Multiple Experts

Socho ek doctor, ek lawyer, aur ek engineer ek hi problem dekh rahe hain — teeno alag-alag angle se analyze karte hain. Multi-Head Attention bhi aisa karta hai — har head ek alag "perspective" se text ko analyze karta hai, phir sab combine hote hain!

Python — Multi-Head Attention (PyTorch) 
import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 512/8 = 64
        
        # Query, Key, Value weight matrices
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
    
    def scaled_dot_product_attention(self, Q, K, V):
        # Attention Score = softmax(Q·Kᵀ / √dₖ) · V
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        weights = torch.softmax(scores, dim=-1)
        return torch.matmul(weights, V)
    
    def forward(self, x):
        batch_size = x.size(0)
        # Project aur heads mein split karo
        Q = self.W_q(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1,2)
        K = self.W_k(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1,2)
        V = self.W_v(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1,2)
        
        attn = self.scaled_dot_product_attention(Q, K, V)
        attn = attn.transpose(1,2).contiguous().view(batch_size, -1, self.d_model)
        return self.W_o(attn)

# Usage
mha = MultiHeadAttention(d_model=512, num_heads=8)
x = torch.randn(2, 10, 512)  # batch=2, seq_len=10, d_model=512
output = mha(x)
print("Output shape:", output.shape)  # torch.Size([2, 10, 512])

2.5 Positional Encoding

📍 Positional Encoding — Order Batao!

Transformer mein words parallel process hote hain — isliye usse nahi pata ki kaun sa word pehle hai aur kaun baad mein! Positional Encoding is problem ko solve karta hai. Har position ko ek unique mathematical signal assign kiya jaata hai.

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
Jahan pos = position, i = dimension index, d_model = model dimension
💡Analogy — GPS Coordinates

Positional encoding bilkul GPS coordinates ki tarah hai. Jaise GPS bata deta hai ki "Main Mumbai ke is specific point par hoon", waise Positional Encoding bata deta hai ki "Main sentence ka 3rd word hoon." Model in coordinates ko word embeddings mein add karke position information encode karta hai.

3.1 Tokens & Tokenization in LLMs

🎫 Tokens — LLM ki Currency

LLMs ke liye token wo basic unit hai jis par sab kuch depend karta hai — pricing, speed, context window, sab kuch! Ek rough rule hai: 1 token ≈ 0.75 words (English mein).

⚠️Hindi/Hinglish Tokenization Warning

Hindi ya Devanagari text mein zyada tokens lagte hain! Ek Hindi word ke liye 3-5 tokens chhoot sakte hain, jabki English mein 1-2 hote hain. Iska matlab hai: Hindi chatbot banane par 3-4x zyada API cost! Isliye production mein always token count monitor karo.

3.2 Embeddings

🌐 Embeddings — Meaning ko Numbers Mein Convert Karo

Embedding ek aisi technique hai jo words ya sentences ko high-dimensional numerical vectors mein convert karti hai. In vectors mein words ke semantic relationships encode hote hain.

💡Famous Analogy

Word2Vec ke famous result: King - Man + Woman ≈ Queen
Yeh isliye possible hua kyunki embeddings space mein relationships preserve hote hain. "King" aur "Queen" ke vectors mein jo difference hai, wahi "Man" aur "Woman" ke beech bhi hai!

Python — OpenAI Embeddings API 
from openai import OpenAI
import numpy as np

client = OpenAI()

def get_embedding(text):
    return client.embeddings.create(
        input=text,
        model="text-embedding-3-small"
    ).data[0].embedding

def cosine_similarity(v1, v2):
    # Cosine similarity: dot product / (magnitudes ka product)
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# Test sentences
s1 = "I love machine learning"
s2 = "AI aur deep learning mujhe bahut pasand hai"  
s3 = "I enjoy cooking biryani"

e1, e2, e3 = get_embedding(s1), get_embedding(s2), get_embedding(s3)

print(f"s1 vs s2 (similar topic): {cosine_similarity(e1,e2):.4f}")
# Output: 0.8234 (high — dono AI ke baare mein hain)

print(f"s1 vs s3 (different topic): {cosine_similarity(e1,e3):.4f}")
# Output: 0.2156 (low — ek AI hai, doosra cooking)

print(f"Embedding dimension: {len(e1)}")
# Output: 1536

3.4 Hallucinations

👻 Hallucinations — Jhootha Yakeen se Bachna

LLM Hallucination tab hoti hai jab model confident tone mein galat ya made-up information present karta hai. Yeh LLMs ki sabse badi problem hai!

Hallucination ke Types

📚
Factual Hallucination

Galat facts: "Eiffel Tower 1920 mein bana" (actually 1889)

🔗
Source Hallucination

Fake references: Non-existent papers ya books cite karna

🧮
Math Hallucination

Galat calculations: "17×23 = 401" (actually 391)

🏥
Medical Hallucination

Dangerous — galat doses ya treatments suggest karna

Hallucination Kyun Hoti Hai?

1
Training data ka pattern completion

LLM actually "predict next token" karta hai — isse kabhi kabhi plausible-sounding lekin galat patterns generate ho jaate hain.

2
Knowledge cutoff

Model ke training data ke baad ki events ke baare mein model ko kuch pata nahi — phir bhi guess karta hai.

3
Rare knowledge

Jo topics training mein kam baar aaye the, unpe model zyada hallucinate karta hai.

4
Overconfidence

Model ki training mein "I don't know" jaisi responses ko penalize kiya gaya hoga.

💡Hallucination Reduce Karne ke Tips

1. RAG use karo — factual data provide karo context mein
2. Grounding instructions — "Answer only from given context"
3. Temperature kam karo — creativity reduce, factual accuracy increase
4. Chain-of-Thought — step by step sochne se logical errors kam hote hain
5. Self-verification — model se apna answer verify karwao

3.5 Temperature, Top-k, Top-p, Sampling

🌡️ Temperature aur Sampling — Creativity Control

Jab LLM next token predict karta hai, toh usse probability distribution milta hai — har possible next token ki probability. Temperature, Top-k, Top-p — ye parameters decide karte hain ki us distribution se kaisa token sample karein.

Temperature — Creativity Knob

TemperatureEffectUse CaseExample
0.0Always most likely token (deterministic)Factual Q&A, Code"1+1 = 2" always
0.3-0.5Slightly creative, mostly focusedSummaries, Data extractionConsistent but varied
0.7-0.9Balanced creativityChat, Content writingNatural conversation
1.0+Highly creative, sometimes randomPoetry, Creative writingUnexpected metaphors
2.0+Near-random, incoherentNot recommendedGibberish possible
Temperature Effect on Token Probability Distribution

Top-k Sampling

Top-k sampling mein sirf top k most probable tokens ko consider kiya jaata hai, baaki sab ignore. Jaise k=50 means: sirf 50 most likely next words mein se sample karo.

Top-p (Nucleus) Sampling

Top-p mein cumulative probability threshold p use hota hai. Jab tak top tokens ki cumulative probability ≤ p ho, unhe consider karo. Yeh zyada adaptive hai Top-k se.

Top-p: Sum tokens until Σ P(token) ≥ p (e.g., 0.9)
p=0.9 means: Top tokens jo milke 90% probability cover karte hain, unhi mein se sample karo
Python — Temperature & Sampling Parameters with OpenAI 
from openai import OpenAI

client = OpenAI()

def generate_text(prompt, temperature=0.7, top_p=1.0):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=temperature,  # 0.0 - 2.0
        top_p=top_p,            # 0.1 - 1.0
        max_tokens=100
    )
    return response.choices[0].message.content

prompt = "Write a one-line story about AI"

# Low temperature — consistent, focused
print("Temp 0.1:", generate_text(prompt, temperature=0.1))
# Output: "An AI learned to understand human emotions."
# (Almost same output every run)

# High temperature — creative, varied
print("Temp 1.5:", generate_text(prompt, temperature=1.5))
# Output: "The last algorithm dreamed in electric verse."
# (Different every run — more creative!)

# Top-p sampling
print("Top-p 0.5:", generate_text(prompt, temperature=1.0, top_p=0.5))
# More focused — only top 50% probable tokens considered

4.2 Quantization

📦 Quantization — Size Chhota Karo, Speed Badhao

LLMs mein billions of parameters hote hain, jo originally FP32 (32-bit floating point) mein store hote hain. Quantization in parameters ko lower precision (8-bit, 4-bit, even 2-bit) mein convert karta hai — size aur memory dramatically reduce ho jaati hai!

✅ Quantization ke Fayde
  • Model size 4-8x chhota hota hai
  • Inference speed 2-4x faster
  • GPU memory kam chahiye
  • Consumer hardware par bhi run ho sakta hai
  • Cloud costs significantly kam
❌ Quantization ke Nuksan
  • Thodi accuracy loss (usually minimal)
  • Complex tasks mein quality thodi drop
  • Integer overflow edge cases
  • 2-bit jaise extreme cases degrade karte hain

4.6 LoRA & PEFT

🔧 LoRA aur PEFT — Smart Fine-tuning

Full fine-tuning mein problem ye hai ki millions of GPU memory chahiye aur sab parameters update hote hain — bahut expensive! PEFT (Parameter Efficient Fine-Tuning) methods sirf kuch parameters update karte hain baaki frozen rakhte hain.

LoRA (Low-Rank Adaptation) — Smart Mathematics

W' = W + ΔW = W + BA
W = original frozen weight matrix (large) | B, A = low-rank matrices (tiny, trainable)
💡LoRA Intuition

Socho ek 1000×1000 matrix ko update karna padta hai = 1,000,000 parameters.
LoRA kehta hai: Ise 1000×r aur r×1000 matrices se represent karo (jahan r=8 ya 16).
Ab sirf 2×1000×8 = 16,000 parameters train karne padenge — 62x less!
Full model quality ka 95%+ quality milti hai, cost fraction mein!

LoRA — Weight Update Visualization
Python — LoRA Fine-tuning with PEFT + TRL 
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model, TaskType
from trl import SFTTrainer
from datasets import load_dataset

# 1. Base model load karo
model_id = "mistralai/Mistral-7B-v0.1"
model = AutoModelForCausalLM.from_pretrained(model_id)
tokenizer = AutoTokenizer.from_pretrained(model_id)

# 2. LoRA configuration define karo
lora_config = LoraConfig(
    r=16,                    # Rank — small = less params, fast
    lora_alpha=32,             # Scaling factor (2*r is common)
    target_modules=["q_proj", "v_proj"],  # Attention layers
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM
)

# 3. LoRA apply karo
peft_model = get_peft_model(model, lora_config)

# Trainable parameters check karo
peft_model.print_trainable_parameters()
# Output: trainable params: 4,194,304 || all params: 7,246,419,968 (0.06% trainable!)

# 4. Dataset aur Trainer setup
dataset = load_dataset("your_custom_dataset", split="train")

trainer = SFTTrainer(
    model=peft_model,
    train_dataset=dataset,
    dataset_text_field="text",
    tokenizer=tokenizer,
    max_seq_length=512
)

trainer.train()
peft_model.save_pretrained("./my-fine-tuned-lora")

🔁 Quick Revision — Part 5 ke Saare Important Points

🔤
Tokenization

BPE = modern standard. 1 token ≈ 0.75 words (English). Hindi mein 3-4x zyada tokens!

✂️
Stemming vs Lemma

Stemming = fast but rough. Lemmatization = accurate, grammar-correct. SpaCy for lemma.

👁️
Attention Formula

Attention(Q,K,V) = softmax(Q·Kᵀ/√dₖ)·V. Q=query, K=key, V=value.

🎭
Multi-Head

8-16 parallel heads. Har head alag relationship pakadta hai. Concatenate + project.

🔄
Encoder vs Decoder

Encoder=understanding (BERT). Decoder=generation (GPT). Both=translation (T5).

📍
Positional Encoding

Sin/Cos signals. Without this, model ko word order pata nahi. Input embedding mein add.

🌐
Embeddings

King - Man + Woman ≈ Queen. Similar meaning = close vectors. 1536 dims for OpenAI.

🪟
Context Window

GPT-4=128K, Claude=200K, Gemini=1M. Full hone par model bhool jaata hai!

👻
Hallucinations

Confident but wrong. RAG + grounding + low temperature se reduce karo.

🌡️
Temperature

0=same always. 0.7=balanced. 1.5+=creative. Top-p=nucleus sampling.

🎪
MoE

Router selects top-2 experts. Sparse activation. Mixtral = MoE based. 3-4x faster.

📦
Quantization

FP32→INT4 = 8x smaller. Q4 ≈ 95% quality. bitsandbytes library use karo.

👥
RLHF

3 steps: SFT → Reward Model → PPO. Human feedback se alignment. ChatGPT ka secret.

🔧
LoRA

W' = W + BA. r=16 = 0.06% trainable params. 1750x less than full fine-tuning!

🧪
Distillation

Teacher → Student. DistilBERT = 60% faster, 97% quality. Soft labels better hain.
