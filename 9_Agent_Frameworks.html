Chapter 1 — Agent Frameworks kya hote hain?

Jab hum ek AI Agent banana chahte hain — jaise ek research assistant jo internet search kare, documents padhe, code likhe, aur apne aap decisions le — toh hume bahut saari cheezein manage karni padti hain: LLM calls, memory, tools, planning, error handling, aur state management.

Agar hum yeh sab scratch se likhein, toh bahut time waste hoga. Isliye Agent Frameworks aate hain — ye ready-made building blocks provide karte hain jinse hum quickly powerful agents bana sakte hain.

📘 Simple Definition
Agent Framework ek software library hai jo AI Agents banane ke liye pre-built components provide karta hai — jaise LLM integration, memory management, tool calling, multi-agent coordination, aur workflow orchestration. Yeh frameworks "Agent ka skeleton" hote hain — tum sirf logic dalo, framework sab handle karta hai.

Agent Framework ki zaroorat kyun hai?

1
🔧 Boilerplate Code reduce karta hai
Bina framework ke, har bar LLM call, error handling, retry logic likhna padta. Framework yeh sab handle karta hai.
2
🧩 Modularity deta hai
Memory, Tools, LLM — sab alag-alag components hote hain. Aap ek swap kar sakte ho bina baaki ko todne ke.
3
🚀 Production patterns built-in hote hain
Streaming, async execution, observability, tracing — yeh sab frameworks me already integrated hoti hain.
4
🌐 Ecosystem integration
OpenAI, Anthropic, vector databases, search APIs — frameworks inhe easily connect karte hain bina extra code ke.
5
👥 Community & Plugins
Thousands of contributors ne plugins, integrations, examples banaye hain. Aapko wheel reinvent nahi karna.

Framework Selection Mindset

Framework choose karte waqt yeh questions pucho:

  • Kya mujhe simple chains chahiye ya complex stateful workflows? → LangChain vs LangGraph
  • Kya mera task mainly document retrieval / RAG hai? → LlamaIndex
  • Kya mujhe multiple AI agents milkar kaam karte chahiye? → CrewAI ya AutoGen
  • Kya mera organization Microsoft stack use karta hai? → Semantic Kernel
  • Kya mujhe production-grade NLP pipeline chahiye? → Haystack

1. Chains — Kya hain aur kaise kaam karte hain?

Chain ek pipeline hai jahan ek step ka output directly doosre step ka input banta hai. Jaise assembly line mein har station ek kaam karta hai — chain mein bhi ऐसा hi hota hai.

Python — Simple Chain (LangChain)
from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Step 1: Prompt Template banao prompt = ChatPromptTemplate.from_messages([ ("system", "Tum ek helpful AI assistant ho."), ("user", "{question}") ]) # Step 2: LLM initialize karo llm = ChatOpenAI(model="gpt-4o", temperature=0.7) # Step 3: Output parser parser = StrOutputParser() # Step 4: Chain banao — LCEL syntax (pipe operator) chain = prompt | llm | parser # Step 5: Chain invoke karo result = chain.invoke({"question": "Python kya hai?"}) print(result) # Output: "Python ek high-level, interpreted programming language hai..."

LangChain ke Key Components

1. LCEL — LangChain Expression Language

LCEL LangChain ka modern way hai components ko compose karne ka. Pipe operator (|) se chains banate hain — bilkul Unix commands ki tarah.

Python — LCEL Complete Example
from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser, JsonOutputParser from langchain_core.runnables import RunnableParallel, RunnablePassthrough # === Basic Chain === basic_chain = ( ChatPromptTemplate.from_template("Topic: {topic} ke baare mein batao") | ChatOpenAI(model="gpt-4o-mini") | StrOutputParser() ) # === Parallel Chains — dono kaam ek saath === parallel_chain = RunnableParallel( summary=basic_chain, keywords=( ChatPromptTemplate.from_template( "{topic} ke 5 keywords JSON mein do: {{keywords: [...]}}" ) | ChatOpenAI() | JsonOutputParser() ) ) # Invoke karo result = parallel_chain.invoke({"topic": "Machine Learning"}) print(result["summary"]) # ML ki summary print(result["keywords"]) # {"keywords": ["neural", "training", ...]} # === Streaming support === for chunk in basic_chain.stream({"topic": "AI Agents"}): print(chunk, end="", flush=True)

2. LangChain Agents with Tools

Python — Tool-Using Agent
from langchain_openai import ChatOpenAI from langchain.agents import create_tool_calling_agent, AgentExecutor from langchain_core.tools import tool from langchain_core.prompts import ChatPromptTemplate # Custom Tool banana — decorator use karo @tool def calculate_area(length: float, width: float) -> float: """Kisi bhi rectangle ka area calculate karta hai.""" return length * width @tool def get_weather(city: str) -> str: """Kisi city ka weather batata hai.""" # Real implementation mein API call hogi return f"{city} mein aaj 28°C, sunny weather hai" # Tools list tools = [calculate_area, get_weather] # Agent prompt prompt = ChatPromptTemplate.from_messages([ ("system", "Tum ek helpful assistant ho. Diye gaye tools use karo."), ("placeholder", "{chat_history}"), ("human", "{input}"), ("placeholder", "{agent_scratchpad}"), ]) # LLM + Agent + Executor llm = ChatOpenAI(model="gpt-4o") agent = create_tool_calling_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # Agent ko kaam do result = executor.invoke({ "input": "Mumbai ka weather kya hai aur 10x5 ka area kya hoga?", "chat_history": [] }) # Output: "Mumbai mein 28°C hai aur area 50 square units hai"

3. RAG with LangChain

Python — RAG Pipeline with LangChain
from langchain_openai import ChatOpenAI, OpenAIEmbeddings from langchain_community.vectorstores import Chroma from langchain_community.document_loaders import PyPDFLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain_core.prompts import ChatPromptTemplate from langchain_core.runnables import RunnablePassthrough from langchain_core.output_parsers import StrOutputParser # Step 1: Document load karo loader = PyPDFLoader("my_document.pdf") docs = loader.load() # Step 2: Chunks mein todo splitter = RecursiveCharacterTextSplitter( chunk_size=500, chunk_overlap=50 ) chunks = splitter.split_documents(docs) # Step 3: Vector store mein store karo vectorstore = Chroma.from_documents( chunks, OpenAIEmbeddings() ) retriever = vectorstore.as_retriever(search_kwargs={"k": 3}) # Step 4: RAG Prompt rag_prompt = ChatPromptTemplate.from_template(""" Context: {context} Question: {question} Answer the question using ONLY the context above. """) # Step 5: Complete RAG Chain rag_chain = ( {"context": retriever, "question": RunnablePassthrough()} | rag_prompt | ChatOpenAI() | StrOutputParser() ) # Query karo print(rag_chain.invoke("Document mein kya likha hai?"))

Chapter 4 — LangGraph 🕸

Intermediate Advanced Production Ready
🕸
LangGraph
Stateful, multi-actor applications — Graph-based agent orchestration

LangGraph LangChain team ka ek powerful extension hai jo complex, stateful workflows banane deta hai. Jahan LangChain linear chains banata hai, LangGraph cyclic graphs banata hai — matlab agent loop mein kaam kar sakta hai, decisions le sakta hai, aur state maintain kar sakta hai.

LangGraph vs LangChain — Kab Kya Use Karein?

FeatureLangChainLangGraph
Flow TypeLinear / DAGCyclic Graphs (loops supported)
State ManagementLimitedFirst-class — TypedDict based
Human-in-loopDifficultBuilt-in (interrupt, breakpoints)
Conditional RoutingBasicPowerful conditional edges
DebuggingCallbacksLangSmith Studio visual debugger
Use CaseSimple agents, RAGComplex multi-step agents, workflows

🕸 LangGraph — ReAct Agent Graph Structure

LangGraph Complete Code Example

Python — LangGraph ReAct Agent
from typing import TypedDict, Annotated from langgraph.graph import StateGraph, START, END from langgraph.prebuilt import ToolNode, tools_condition from langchain_openai import ChatOpenAI from langchain_core.tools import tool from langgraph.graph.message import add_messages # === Step 1: State define karo === class AgentState(TypedDict): # messages list — Annotated[list, add_messages] se auto-merge hoga messages: Annotated[list, add_messages] # === Step 2: Tools define karo === @tool def search_web(query: str) -> str: """Web pe search karta hai kisi topic ke baare mein.""" return f"Search result for '{query}': Sample data..." tools = [search_web] # === Step 3: LLM ko tools bind karo === llm = ChatOpenAI(model="gpt-4o").bind_tools(tools) # === Step 4: Nodes define karo === def agent_node(state: AgentState): """Main agent jo LLM call karta hai""" response = llm.invoke(state["messages"]) return {"messages": [response]} # ToolNode automatically tools execute karta hai tool_node = ToolNode(tools) # === Step 5: Graph banao === graph = StateGraph(AgentState) # Nodes add karo graph.add_node("agent", agent_node) graph.add_node("tools", tool_node) # Edges add karo graph.add_edge(START, "agent") graph.add_conditional_edges( "agent", tools_condition, # tool_calls hai → "tools", warna → END ) graph.add_edge("tools", "agent") # tool ke baad wapas agent # === Step 6: Compile & Run === app = graph.compile() result = app.invoke({ "messages": [{"role": "user", "content": "AI agents kya hote hain?"}] }) print(result["messages"][-1].content)

Human-in-the-Loop with LangGraph

Python — Human Approval Checkpoint
from langgraph.checkpoint.memory import MemorySaver # Checkpointer — state save karta hai memory = MemorySaver() # Compile with interrupt — tool se pehle human approval app = graph.compile( checkpointer=memory, interrupt_before=["tools"] # tool run se PEHLE ruko ) config = {"configurable": {"thread_id": "session_1"}} # Step 1: Pehli run — tool call pe ruk jaega result = app.invoke({ "messages": [{"role": "user", "content": "Search AI trends"}] }, config=config) # Step 2: Human check karo state = app.get_state(config) print("Agent iska tool use karna chahta hai:") print(state.values["messages"][-1].tool_calls) # Step 3: Approve karo — None input se continue hoga approval = input("Approve? (y/n): ") if approval == "y": final = app.invoke(None, config=config) print(final["messages"][-1].content)
🧠 Remember This — LangGraph
LangGraph ka power uski state management aur cycles mein hai. Jab bhi tumhara agent ko loop karna ho, state maintain karna ho, ya human approval chahiye ho — LangGraph use karo. Simple linear tasks ke liye LangChain kaafi hai.

Python — LlamaIndex Basic RAG
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader from llama_index.core import Settings from llama_index.llms.openai import OpenAI from llama_index.embeddings.openai import OpenAIEmbedding # Settings configure karo (global) Settings.llm = OpenAI(model="gpt-4o", temperature=0.1) Settings.embed_model = OpenAIEmbedding() # Step 1: Documents load karo documents = SimpleDirectoryReader("./my_docs/").load_data() print(f"Loaded {len(documents)} documents") # Step 2: Index banao — automatically chunks & embeds index = VectorStoreIndex.from_documents(documents) # Step 3: Query Engine create karo query_engine = index.as_query_engine( similarity_top_k=3 # Top 3 relevant chunks ) # Step 4: Query karo response = query_engine.query("Document mein main topic kya hai?") print(response) print("\n--- Source Nodes ---") for node in response.source_nodes: print(f"Score: {node.score:.2f} | Text: {node.text[:100]}...")
Python — LlamaIndex Agent with Tools
from llama_index.core.agent import ReActAgent from llama_index.core.tools import FunctionTool, QueryEngineTool # === Tool 1: Custom function tool === def multiply_numbers(a: float, b: float) -> float: """Do numbers ko multiply karta hai.""" return a * b math_tool = FunctionTool.from_defaults(fn=multiply_numbers) # === Tool 2: Index ko tool banana === doc_tool = QueryEngineTool.from_defaults( query_engine=query_engine, name="document_search", description="Documents mein information search karta hai." ) # === ReAct Agent banao === agent = ReActAgent.from_tools( [math_tool, doc_tool], verbose=True ) # Query karo response = agent.chat("Document mein bataye price ko 1.18 se multiply karo") print(response)

CrewAI
Role-based multi-agent framework — Agents ki "crew" milkar kaam karti hai

CrewAI ek framework hai jahan aap multiple AI agents ko ek team ki tarah organize kar sakte ho. Har agent ka ek specific role, goal, aur backstory hota hai — jaise ek real company mein CEO, Developer, aur Designer hote hain. Yeh "AI ke workforce" banane ka sabse intuitive framework hai.

Python — CrewAI Complete Example
from crewai import Agent, Task, Crew, Process from crewai_tools import SerperDevTool from langchain_openai import ChatOpenAI # LLM llm = ChatOpenAI(model="gpt-4o") search_tool = SerperDevTool() # === Step 1: Agents define karo === researcher = Agent( role="Senior Research Analyst", goal="AI industry ke latest trends dhundho aur accurate data provide karo", backstory="""Tum ek experienced research analyst ho jo 10 saal se technology trends analyze kar raha hai. Tumhara analysis always data-driven aur comprehensive hota hai.""", tools=[search_tool], llm=llm, verbose=True ) writer = Agent( role="Content Strategist", goal="Research ko ek engaging aur educational article mein convert karo", backstory="""Tum ek master storyteller ho jo complex technical topics ko easy language mein explain karna jaante ho.""", llm=llm, verbose=True ) # === Step 2: Tasks define karo === research_task = Task( description="""2024-2025 mein top 5 AI Agent frameworks ke baare mein comprehensive research karo. Include: popularity, use cases, pros/cons.""", expected_output="Detailed research report with data and sources", agent=researcher ) write_task = Task( description="""Research ko use karke ek beginner-friendly blog post likho jo explain kare 'Which AI Framework Should You Choose in 2025?'""", expected_output="800 word blog post in markdown format", agent=writer, context=[research_task] # Research ka output use karega ) # === Step 3: Crew banao === crew = Crew( agents=[researcher, writer], tasks=[research_task, write_task], process=Process.sequential, # Ek ke baad ek verbose=True ) # === Step 4: Crew ko kaam pe lagao === result = crew.kickoff() print(result.raw)

Python — AutoGen Basic Agent Conversation
import autogen # Configuration config_list = [{ "model": "gpt-4o", "api_key": "your-api-key" }] llm_config = {"config_list": config_list} # === AssistantAgent — LLM powered === assistant = autogen.AssistantAgent( name="AI_Assistant", llm_config=llm_config, system_message="""Tum ek expert Python developer ho. Sada working code do aur explain karo.""" ) # === UserProxyAgent — Code execute karta hai === user_proxy = autogen.UserProxyAgent( name="User_Proxy", human_input_mode="NEVER", # Auto — human se input nahi lega max_consecutive_auto_reply=10, is_termination_msg=lambda x: x.get("content", "").rstrip().endswith("TERMINATE"), code_execution_config={ "work_dir": "coding_workspace", "use_docker": False } ) # === Conversation start karo === user_proxy.initiate_chat( assistant, message=""" Ek Python script banao jo: 1. 1 se 100 tak ke prime numbers find kare 2. Result plot kare matplotlib se 3. CSV mein save kare Reply TERMINATE when done. """ )

AutoGen GroupChat — Multiple Agents

Python — AutoGen GroupChat
import autogen # Multiple agents define karo coder = autogen.AssistantAgent( name="Coder", llm_config=llm_config, system_message="Tum ek expert coder ho. Sirf code likho." ) reviewer = autogen.AssistantAgent( name="Code_Reviewer", llm_config=llm_config, system_message="Tum code review expert ho. Bugs aur improvements batao." ) user_proxy = autogen.UserProxyAgent( name="User", human_input_mode="TERMINATE", code_execution_config={"work_dir": "."} ) # GroupChat setup groupchat = autogen.GroupChat( agents=[user_proxy, coder, reviewer], messages=[], max_round=12 ) manager = autogen.GroupChatManager( groupchat=groupchat, llm_config=llm_config ) # Start karo user_proxy.initiate_chat( manager, message="Ek sorting algorithm implement karo aur review karo" )

Python — LangGraph Supervisor Multi-Agent
from typing import TypedDict, Annotated, Literal from langgraph.graph import StateGraph, START, END from langgraph.graph.message import add_messages from langchain_openai import ChatOpenAI # State class TeamState(TypedDict): messages: Annotated[list, add_messages] next_agent: str llm = ChatOpenAI(model="gpt-4o") # Supervisor — decide karta hai kisko kaam dena hai def supervisor_node(state): msgs = state["messages"] system = """Tum supervisor ho. Team: researcher, coder, writer. Decide karo next me kaunsa agent kaam karega. Sirf ek naam do: researcher/coder/writer/FINISH""" response = llm.invoke([{"role": "system", "content": system}] + msgs) return {"next_agent": response.content.strip()} # Worker nodes def researcher_node(state): response = llm.invoke([{"role": "system", "content": "Tum research specialist ho."}] + state["messages"]) return {"messages": [{"role": "assistant", "content": f"[Researcher]: {response.content}"}]} def coder_node(state): response = llm.invoke([{"role": "system", "content": "Tum expert coder ho."}] + state["messages"]) return {"messages": [{"role": "assistant", "content": f"[Coder]: {response.content}"}]} # Routing function def route_to_agent(state) -> Literal["researcher", "coder", "__end__"]: next_a = state.get("next_agent", "") if "FINISH" in next_a: return "__end__" if "researcher" in next_a: return "researcher" return "coder" # Graph build karo graph = StateGraph(TeamState) graph.add_node("supervisor", supervisor_node) graph.add_node("researcher", researcher_node) graph.add_node("coder", coder_node) graph.add_edge(START, "supervisor") graph.add_conditional_edges("supervisor", route_to_agent) graph.add_edge("researcher", "supervisor") graph.add_edge("coder", "supervisor") app = graph.compile() result = app.invoke({ "messages": [{"role": "user", "content": "AI agent frameworks research karke Python code likho"}], "next_agent": "" })
⚠️ Multi-Agent Common Mistakes
  • Infinite loops: Termination condition define nahi ki — agents chaklate rahte hain
  • Context overflow: Bahut saari agents ke messages se token limit exceed ho jaati hai
  • Agent hallucinations: Ek agent galat info doosre ko pass karta hai — compounding errors
  • No error handling: Ek agent fail ho toh poora workflow crash ho jaata hai
