🌟 Deep Learning Kya Hai?

Deep Learning, Machine Learning ka ek powerful branch hai jisme hum artificial neural networks use karte hain jo human brain ke neurons se inspired hain. "Deep" word isliye aata hai kyunki in networks mein bahut saari layers hoti hain — jitni zyada layers, utna "deep" network.

Socho ek pyaaz (onion) ki layers ki tarah — har layer pehli layer se kuch aur seekhti hai. Pehli layer simple cheezein detect karti hai (jaise lines), baad ki layers complex cheezein (jaise faces, objects) detect karti hain.

📘 Key Insight

Deep Learning ne 2012 ke baad AI field ko revolutionize kiya jab AlexNet ne ImageNet competition mein humans se better performance di. Tab se image recognition, speech, NLP — sabka paradigm badal gaya.

Hamara brain lगभग 86 billion neurons se bana hai. Har neuron doosre neurons se connected hota hai. Jab koi signal aata hai, neuron decide karta hai — activate karna hai ya nahi.

Artificial neuron isi concept ko mathematically model karta hai:

  • Cell Body — signals process karta hai

  • Weights (w) — input ki importance

⚡ Activation Functions

Activation function decide karti hai ki ek neuron ko "fire" karna chahiye ya nahi. Bina activation function ke, neural network sirf ek linear model hoga — chaahe kitni bhi layers ho.

⚠️ Kyun Zaroori Hai?

Agar activation function nahi ho toh sabhi layers ka combination ek hi linear transformation ban jaata hai: y = Wx + b — aur hum deep learning ka faida nahi utha sakte! Non-linearity introduce karna activation functions ka kaam hai.

Sigmoid
σ(x) = 1/(1+e⁻ˣ)
Output: 0 to 1. Binary classification ke liye use hota hai. Vanishing gradient problem aati hai deep networks mein.
Classification
Tanh
tanh(x) = (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ)
Output: -1 to 1. Sigmoid se better — zero-centered. RNN mein commonly used.
RNN/LSTM
ReLU ⭐
f(x) = max(0, x)
Sabse popular! Fast, simple. Negative values 0 ho jaate hain. "Dying ReLU" problem hoti hai.
Most Common
Leaky ReLU
f(x) = max(0.01x, x)
ReLU ka improvement. Negative side pe small slope deta hai — dying ReLU fix karta hai.
Better ReLU
Softmax
σ(z)ⱼ = eᶻʲ / Σeᶻᵏ
Multi-class classification ke output layer mein. Probabilities deta hai (sum = 1).
Output Layer
GELU
0.5x(1+tanh(√(2/π)(x+0.044715x³)))
Transformers mein use hota hai (BERT, GPT). ReLU se smoother behavior.

➡️ Forward Propagation

Forward propagation woh process hai jisme data input se output ki taraf flow karta hai. Step by step samajhte hain ek simple example ke saath:

  • 1
    Input Receive Karo Feature vector X = [x₁, x₂, ..., xₙ] as input. Jaise ek image ke 784 pixel values.
  • 2
    Weighted Sum Calculate Karo (z) z = W·X + b (weights ke saath multiply karke sum karo, phir bias add karo)
  • 3
    Activation Apply Karo a = σ(z) — activation function se output produce karo
  • 4
    Next Layer Ko Pass Karo Current layer ka output, next layer ka input ban jaata hai
  • 5
    Final Output Generate Karo Last layer se prediction aati hai: class probabilities ya regression value
Python — Forward Propagation (Manual)
import numpy as np

# ========================================
# Forward Propagation — Step by Step
# ========================================

# Input: 3 features ka ek sample
X = np.array([0.5, 0.8, 0.3])

# Layer 1 weights (3 inputs → 4 neurons)
W1 = np.array([
    [0.1, 0.4, 0.7, 0.2],
    [0.3, 0.6, 0.1, 0.5],
    [0.8, 0.2, 0.4, 0.9]
])
b1 = np.array([0.1, 0.2, 0.1, 0.3])

# Layer 2 weights (4 inputs → 2 outputs)
W2 = np.array([
    [0.5, 0.3],
    [0.2, 0.8],
    [0.6, 0.1],
    [0.4, 0.7]
])
b2 = np.array([0.1, 0.2])

# Activation functions
def relu(z):
    return np.maximum(0, z)

def softmax(z):
    exp_z = np.exp(z - np.max(z))  # stability ke liye
    return exp_z / np.sum(exp_z)

# === FORWARD PASS ===

# Layer 1: z1 = W1ᵀ · X + b1
z1 = np.dot(X, W1) + b1
print(f"z1 (pre-activation): {z1}")

# Layer 1: Activation (ReLU)
a1 = relu(z1)
print(f"a1 (after ReLU): {a1}")

# Layer 2: z2 = W2ᵀ · a1 + b2
z2 = np.dot(a1, W2) + b2
print(f"z2 (pre-activation): {z2}")

# Layer 2: Activation (Softmax)
output = softmax(z2)
print(f"Final Output (probabilities): {output}")
print(f"Predicted Class: {np.argmax(output)}")

# Output:
# z1 (pre-activation): [0.49 0.78 0.65 0.84]
# a1 (after ReLU): [0.49 0.78 0.65 0.84]
# z2 (pre-activation): [1.12 1.58]
# Final Output: [0.387 0.613]
# Predicted Class: 1

⬅️ Backpropagation — The Heart of Learning

Backpropagation Deep Learning ka sabse important algorithm hai. Isse hi neural network "seekhna" possible hota hai. Yeh chain rule of calculus use karke loss ko minimize karne ke liye weights update karta hai.

💡 Intuition

Socho ek exam mein tumne galat answer diya. Teacher teri galti batata hai (loss). Ab tum seedhe teacher ki baat sunte ho, uske teacher ki baat sunte ho (backprop chain), aur apni samajh (weights) update karte ho. Yahi backpropagation hai — error ko peeche propagate karna aur har weight ko update karna.

  • 1
    Forward Pass Karo Pehle normal forward propagation karke prediction lo
  • 2
    Loss Calculate Karo Loss = L(y_true, y_pred) — kitni galti hui?
  • 3
    Output Layer Ka Gradient Nikalo ∂L/∂W_output — loss ke saath respect to output weights
  • 4
    Hidden Layers Mein Propagate Karo (Chain Rule) ∂L/∂W_hidden = ∂L/∂a × ∂a/∂z × ∂z/∂W — chain rule se back jaao
  • 5
    Weights Update Karo (Gradient Descent) W = W - α × ∂L/∂W — learning rate × gradient se weights update
Forward & Backward Pass — Data Flow
Python — Backpropagation (Manual Implementation)
import numpy as np

# Simple 2-layer network: backprop manual
np.random.seed(42)

# Data: XOR problem
X = np.array([[0,0],[0,1],[1,0],[1,1]])
y = np.array([[0],[1],[1],[0]])

# Random weights initialize karo
W1 = np.random.randn(2, 4) * 0.1
b1 = np.zeros((1, 4))
W2 = np.random.randn(4, 1) * 0.1
b2 = np.zeros((1, 1))
lr = 0.1

def sigmoid(z): return 1 / (1 + np.exp(-z))
def sigmoid_deriv(a): return a * (1 - a)

for epoch in range(10000):
    # === FORWARD PASS ===
    z1 = np.dot(X, W1) + b1     # (4,4)
    a1 = sigmoid(z1)             # (4,4)
    z2 = np.dot(a1, W2) + b2    # (4,1)
    a2 = sigmoid(z2)             # (4,1) — final prediction

    # === LOSS (Binary Cross-Entropy) ===
    loss = -np.mean(y * np.log(a2) + (1-y) * np.log(1-a2))

    # === BACKWARD PASS ===
    # Output layer gradient
    dL_da2 = -(y/a2) + ((1-y)/(1-a2))
    da2_dz2 = sigmoid_deriv(a2)
    dL_dz2 = dL_da2 * da2_dz2   # (4,1)

    dL_dW2 = np.dot(a1.T, dL_dz2)  # (4,1)
    dL_db2 = np.sum(dL_dz2, axis=0)

    # Hidden layer gradient (chain rule)
    dL_da1 = np.dot(dL_dz2, W2.T)  # (4,4)
    da1_dz1 = sigmoid_deriv(a1)
    dL_dz1 = dL_da1 * da1_dz1      # (4,4)

    dL_dW1 = np.dot(X.T, dL_dz1)   # (2,4)
    dL_db1 = np.sum(dL_dz1, axis=0)

    # === WEIGHTS UPDATE ===
    W2 -= lr * dL_dW2
    b2 -= lr * dL_db2
    W1 -= lr * dL_dW1
    b1 -= lr * dL_db1

    if epoch % 2000 == 0:
        print(f"Epoch {epoch}: Loss = {loss:.4f}")

# Epoch 0:    Loss = 0.6932
# Epoch 2000: Loss = 0.2541
# Epoch 4000: Loss = 0.1089
# Epoch 6000: Loss = 0.0612
# Epoch 8000: Loss = 0.0421

🔷 ANN — Artificial Neural Network

ANN ya Artificial Neural Network — yeh sabse basic deep learning architecture hai. Isme fully-connected layers hoti hain (har neuron doosre layer ke har neuron se connected). Isko Multi-Layer Perceptron (MLP) bhi kehte hain.

✅ ANN Ka Use Karo Jab:

  • Tabular/structured data ho
  • Simple classification tasks
  • Regression problems
  • Koi spatial/temporal pattern nahi
  • Features clearly defined hain

❌ ANN Avoid Karo Jab:

  • Images ho (CNN better hai)
  • Sequential data ho (RNN better hai)
  • Very high-dimensional raw data
  • Spatial relationships important hain
Python — ANN with PyTorch
import torch
import torch.nn as nn
import torch.optim as optim
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split
import numpy as np

# Data load aur preprocess
iris = load_iris()
X, y = iris.data.astype(np.float32), iris.target

scaler = StandardScaler()
X = scaler.fit_transform(X)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

X_train = torch.FloatTensor(X_train)
X_test  = torch.FloatTensor(X_test)
y_train = torch.LongTensor(y_train)
y_test  = torch.LongTensor(y_test)

# ANN Model Define Karo
class ANN(nn.Module):
    def __init__(self):
        super(ANN, self).__init__()
        self.layers = nn.Sequential(
            nn.Linear(4, 16),   # 4 features → 16 neurons
            nn.ReLU(),
            nn.Linear(16, 8),   # 16 → 8
            nn.ReLU(),
            nn.Linear(8, 3)     # 8 → 3 classes (output)
        )

    def forward(self, x):
        return self.layers(x)

model = ANN()
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.01)

# Training Loop
for epoch in range(200):
    model.train()
    optimizer.zero_grad()
    outputs = model(X_train)
    loss = criterion(outputs, y_train)
    loss.backward()     # Backpropagation!
    optimizer.step()    # Weights update!

    if epoch % 50 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")

# Evaluation
model.eval()
with torch.no_grad():
    predictions = model(X_test)
    predicted_classes = torch.argmax(predictions, dim=1)
    accuracy = (predicted_classes == y_test).float().mean()
    print(f"Test Accuracy: {accuracy:.2%}")

# Output:
# Epoch 0, Loss: 1.2142
# Epoch 50, Loss: 0.3841
# Epoch 100, Loss: 0.1992
# Epoch 150, Loss: 0.1123
# Test Accuracy: 96.67%

🖼️ CNN — Convolutional Neural Network

CNN computer vision ka backbone hai. Ye images, videos aur spatial data ke liye design kiya gaya hai. Iska naam "Convolutional" isliye hai kyunki yeh convolution operation use karta hai.

📘 Real-World Analogy

Socho tum ek photo dekh rahe ho — pehle tumhari aankh edges detect karti hai, phir shapes, phir complete objects. CNN bilkul same karta hai — hierarchically features extract karta hai: edges → textures → shapes → objects.

CNN Ke Main Components

🔍
Convolutional Layer
Filters/kernels image pe slide karte hain aur features extract karte hain. Har filter ek specific pattern (edge, curve) detect karta hai.
📉
Pooling Layer
Spatial dimensions reduce karta hai. Max Pooling sabse popular — har region mein maximum value rakhta hai. Computation reduce + translation invariance.
🔗
Fully Connected Layer
Last mein traditional ANN layers. Flattened features leke final classification/detection karta hai.
🛡️
Batch Normalization
Training stabilize karta hai. Internal covariate shift reduce karta hai. Training bahut fast ho jaati hai.

Python — CNN with PyTorch (MNIST)
import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self):
        super(SimpleCNN, self).__init__()

        # Feature Extraction Block
        self.conv_block = nn.Sequential(
            # Conv1: 1 channel → 32 filters, 3×3 kernel
            nn.Conv2d(1, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),   # 28×28 → 14×14

            # Conv2: 32 → 64 filters
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),   # 14×14 → 7×7

            # Conv3: 64 → 128 filters
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(),
        )

        # Classification Block
        self.classifier = nn.Sequential(
            nn.Flatten(),           # 7×7×128 = 6272 → 1D
            nn.Linear(7 * 7 * 128, 256),
            nn.ReLU(),
            nn.Dropout(0.5),      # Overfitting prevent karo
            nn.Linear(256, 10)    # 10 digits (0-9)
        )

    def forward(self, x):
        x = self.conv_block(x)
        x = self.classifier(x)
        return x

# Model ka summary dekho
model = SimpleCNN()
print(model)

# Total parameters count karo
total_params = sum(p.numel() for p in model.parameters())
print(f"Total Parameters: {total_params:,}")

🔄 RNN — Recurrent Neural Network

RNN sequential/time-series data ke liye design kiya gaya hai. CNN images samajhta hai, RNN sequences samajhta hai — text, audio, time-series, video frames.

📘 Intuition

Socho tum ek story padh rahe ho — "The cat sat on the..." — agli word predict karne ke liye tumhe pichle words yaad rakhne chahiye. RNN exactly yahi karta hai — ek hidden state maintain karta hai jo "memory" ki tarah kaam karta hai.

RNN — Unrolled Through Time
🚨 RNN Ki Problem — Vanishing Gradient

RNN mein ek badi problem hai: vanishing gradient. Jab sequence bahut lamba hota hai (jaise 100+ words), toh backpropagation mein gradients bahut chhote ho jaate hain — practically zero. Iska matlab hai ki network long-term dependencies nahi seekh paata. "The cat, which was very hungry and had not eaten for three days, sat" — yahan "cat" aur "sat" ka connection RNN nahi seekh paata. LSTM isko fix karta hai!

🧬 LSTM — Long Short-Term Memory

LSTM ko 1997 mein Hochreiter & Schmidhuber ne design kiya. Yeh RNN ka advanced version hai jo long-term dependencies ko handle kar sakta hai. Iska secret hai — gates!

🚪
Forget Gate
Decide karta hai ki cell state se kya "bhoolna" hai. Sigmoid output 0 (forget) se 1 (keep) tak hota hai.
📥
Input Gate
Naya information cell state mein add karne ka decision. Sigmoid + Tanh combination use hota hai.
📤
Output Gate
Cell state ka kaun sa part output (hidden state) mein jaayega. Next step ke liye hidden state produce karta hai.
📜
Cell State
LSTM ki "memory highway" — long-term information store karta hai. Gradients iske through freely flow karte hain.
fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf) ← Forget Gate
iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi) ← Input Gate
C̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc) ← Candidate values
Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ ← Cell State update
oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo) ← Output Gate
hₜ = oₜ * tanh(Cₜ) ← Hidden State

⚡ GRU — Gated Recurrent Unit

GRU (2014) LSTM ka simplified version hai. Isme sirf 2 gates hain (Reset + Update) — LSTM ke 3 ke bajaye. GRU zyada efficient hai aur most tasks pe LSTM jaisi hi performance deta hai.

🧬 LSTM vs GRU Comparison

  • Gates: LSTM = 3, GRU = 2
  • Parameters: LSTM > GRU
  • Speed: GRU faster hai
  • Accuracy: Similar on most tasks
  • Memory: GRU less memory use karta hai

📊 Kis Architecture Ka Use Karo?

  • ANN: Tabular data
  • CNN: Images, spatial data
  • RNN: Simple sequences, short text
  • LSTM: Long sequences, speech
  • GRU: Medium sequences, efficient needed
  • Transformer: NLP (modern default)
Python — RNN / LSTM / GRU with PyTorch
import torch
import torch.nn as nn

# =============================================
# Sentiment Analysis — LSTM model
# =============================================

class SentimentLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_layers):
        super().__init__()
        # Word embeddings: word ID → dense vector
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)

        # LSTM layer
        self.lstm = nn.LSTM(
            input_size=embed_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,       # (batch, seq, feature)
            dropout=0.3,            # Regularization
            bidirectional=True      # Forward + Backward
        )

        # Classifier: bidirectional → 2×hidden → output
        self.classifier = nn.Sequential(
            nn.Linear(hidden_dim * 2, 64),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(64, 1),
            nn.Sigmoid()  # Binary: positive/negative
        )

    def forward(self, x):
        embedded = self.embedding(x)     # (batch, seq, embed_dim)
        lstm_out, (hn, cn) = self.lstm(embedded)
        # Last hidden state use karo (both directions)
        hn = torch.cat((hn[-2], hn[-1]), dim=1)  # (batch, 2*hidden)
        out = self.classifier(hn)
        return out.squeeze(1)

# GRU Version (sirf change karo)
class SentimentGRU(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        # GRU — LSTM se simple, fast!
        self.gru = nn.GRU(embed_dim, hidden_dim, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(hidden_dim * 2, 1)

    def forward(self, x):
        embedded = self.embedding(x)
        gru_out, hn = self.gru(embedded)
        hn = torch.cat((hn[-2], hn[-1]), dim=1)
        return torch.sigmoid(self.fc(hn)).squeeze(1)

# Model initialize karo
lstm_model = SentimentLSTM(vocab_size=10000, embed_dim=128, hidden_dim=256, num_layers=2)
gru_model  = SentimentGRU(vocab_size=10000, embed_dim=128, hidden_dim=256)

🔄 Training Loop — Kaise Hota Hai?

⏱️ Epochs, Batch Size, Iterations

📘 Key Definitions

Epoch: Poore training dataset ko ek baar process karna.
Batch Size: Ek saath kitne samples process hote hain.
Iterations: Total samples / Batch size = Ek epoch mein kitne steps.
Example: 1000 samples, batch size 32 → ~31 iterations per epoch.

Batch TypeBatch SizeMemorySpeedGradient Quality
Batch GDFull datasetHighSlowExact, smooth
Stochastic GD (SGD)1Very LowFastNoisy
Mini-batch GD ⭐32–256MediumBalancedGood balance
💡 Best Practice

Industry mein almost always mini-batch gradient descent use hoti hai (batch size 32, 64, 128, 256). Powers of 2 prefer ki jaati hai GPU memory optimization ke liye. Batch size 32 se start karo, phir tune karo.

📐 Gradient Descent — The Optimization Algorithm

Gradient Descent ek optimization algorithm hai jo loss function ko minimize karta hai. Imagine karo tum ek pahaadi pe aankhein band karke neeche utarne ki koshish kar rahe ho — har step mein steepest direction pe paon rakhte ho. Gradient descent yahi karta hai, mathematically.

📏 Learning Rate — Training Ka Most Important Hyperparameter

Learning rate (α) control karta hai ki har step mein weights kitna update honge. Yeh deep learning ka sabse critical hyperparameter hai.

Learning RateEffectSymptomSolution
Too High (e.g., 10.0)Overshoot minimumLoss explodes / divergesReduce LR by 10x
High (e.g., 0.1)Fast but unstableLoss oscillatesReduce LR
Good (e.g., 0.001)Steady convergenceLoss smoothly decreases✅ Keep it
Too Low (e.g., 0.00001)Very slow convergenceStuck, barely improvingIncrease LR / Use LR scheduler

🚀 Optimizers — Gradient Descent Ke Smart Versions

Basic gradient descent slow hoti hai. Modern optimizers iske improved versions hain jo faster convergence aur better results dete hain.

🐢
SGD
Stochastic Gradient Descent. Simple, reliable. Momentum add karo toh better. Deep learning mein mostly Adam prefer hota hai.
🚄
SGD + Momentum
Ball rolling down hill ki tarah — velocity build karta hai. Flat regions cross karna asaan hota hai.
⚡
Adam ⭐
Most popular! Adaptive Moment Estimation — automatically learning rate adjust karta hai. Default choice for most tasks.
🔬
AdamW
Adam + Weight Decay fix. Transformers ke liye preferred. Overfitting reduce karta hai.
📊
RMSprop
Frequently updated parameters ka LR reduce karta hai. RNN training ke liye useful.
🎯
Lion
2023 mein Google Brain ne discover kiya. Large models ke liye Adam se efficient. Sign-based updates.
Adam Optimizer — Update Rules:

mₜ = β₁ · mₜ₋₁ + (1-β₁) · gₜ ← First moment (mean)
vₜ = β₂ · vₜ₋₁ + (1-β₂) · gₜ² ← Second moment (variance)
m̂ₜ = mₜ / (1-β₁ᵗ) ← Bias correction
v̂ₜ = vₜ / (1-β₂ᵗ) ← Bias correction
θₜ = θₜ₋₁ - α · m̂ₜ / (√v̂ₜ + ε) ← Parameter update
Defaults: β₁=0.9, β₂=0.999, ε=1e-8, α=0.001
Python — Complete Training Loop with Best Practices
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

# =============================================
# Production-Ready Training Loop
# =============================================

def train_model(model, train_loader, val_loader, config):
    """
    Complete training loop with:
    - Multiple optimizers support
    - LR scheduler
    - Early stopping
    - Model checkpointing
    """

    # Optimizer choose karo
    optimizer = optim.AdamW(
        model.parameters(),
        lr=config['lr'],
        weight_decay=config['weight_decay']
    )

    # Learning Rate Scheduler — cosine annealing
    scheduler = optim.lr_scheduler.CosineAnnealingLR(
        optimizer,
        T_max=config['epochs'],
        eta_min=1e-6
    )

    criterion = nn.CrossEntropyLoss()
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)

    best_val_loss = float('inf')
    patience_counter = 0
    history = {'train_loss': [], 'val_loss': [], 'val_acc': []}

    for epoch in range(config['epochs']):

        # ---- Training Phase ----
        model.train()
        train_loss = 0.0

        for batch_X, batch_y in train_loader:
            batch_X = batch_X.to(device)
            batch_y = batch_y.to(device)

            optimizer.zero_grad()            # Gradients clear karo
            outputs = model(batch_X)          # Forward pass
            loss = criterion(outputs, batch_y) # Loss compute
            loss.backward()                   # Backpropagation

            # Gradient clipping — exploding gradients rokta hai
            nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

            optimizer.step()                  # Weights update
            train_loss += loss.item()

        train_loss /= len(train_loader)

        # ---- Validation Phase ----
        model.eval()
        val_loss, correct, total = 0.0, 0, 0

        with torch.no_grad():
            for batch_X, batch_y in val_loader:
                batch_X = batch_X.to(device)
                batch_y = batch_y.to(device)
                outputs = model(batch_X)
                loss = criterion(outputs, batch_y)
                val_loss += loss.item()
                _, predicted = torch.max(outputs, 1)
                total += batch_y.size(0)
                correct += (predicted == batch_y).sum().item()

        val_loss /= len(val_loader)
        val_acc = correct / total

        # LR update
        scheduler.step()
        current_lr = scheduler.get_last_lr()[0]

        history['train_loss'].append(train_loss)
        history['val_loss'].append(val_loss)
        history['val_acc'].append(val_acc)

        print(f"Epoch {epoch+1}/{config['epochs']} | "
              f"Train Loss: {train_loss:.4f} | "
              f"Val Loss: {val_loss:.4f} | "
              f"Val Acc: {val_acc:.2%} | "
              f"LR: {current_lr:.6f}")

        # Model Checkpoint — best model save karo
        if val_loss < best_val_loss:
            best_val_loss = val_loss
            torch.save(model.state_dict(), 'best_model.pt')
            patience_counter = 0
            print(f"  ✅ Model saved (val_loss improved)")
        else:
            patience_counter += 1

        # Early Stopping — overfitting rokta hai
        if patience_counter >= config['patience']:
            print(f"⏹️ Early stopping at epoch {epoch+1}")
            break

    return history

# Config
config = {
    'epochs': 100,
    'lr': 0.001,
    'weight_decay': 1e-4,
    'patience': 10  # 10 epochs improvement nahi → stop
}

🛡️ Regularization — Overfitting Ko Rokna

Overfitting tab hota hai jab model training data pe too well fit karta hai par validation/test data pe poorly perform karta hai. Regularization isse rokta hai.

💧
Dropout
Training ke time randomly neurons ko 0 kar deta hai (jaise 50%). Model zyada robust ban jaata hai — kisi single neuron pe depend nahi karta.
⚖️
L2 Regularization
Weight Decay. Loss mein weights ka square add karta hai — very large weights penalize hote hain. AdamW mein built-in hai.
📊
Batch Normalization
Layer outputs ko normalize karta hai. Training stable aur fast hoti hai. Ek built-in regularizer ka kaam karta hai.
📷
Data Augmentation
Artificially training data badhao — flip, rotate, crop, color jitter. Model zyada generalizable ban jaata hai.
⏸️
Early Stopping
Validation loss badhne lage toh training rok do. Overfitting se naturally prevent karta hai.
🎭
Mixup / CutMix
Modern augmentation — do samples mix karo labels ke saath. State-of-the-art results mein use hota hai.

import torch
import torch.nn as nn
import torch.nn.functional as F

# ============================================
# Custom Layer banana — PyTorch mein
# ============================================
class CustomLinear(nn.Module):
    """Apna custom Linear layer"""
    def __init__(self, in_features, out_features):
        super().__init__()
        # Parameters register karo
        self.weight = nn.Parameter(
            torch.randn(in_features, out_features) * 0.01
        )
        self.bias = nn.Parameter(torch.zeros(out_features))

    def forward(self, x):
        return x @ self.weight + self.bias

# ============================================
# ResNet-style Skip Connection Block
# ============================================
class ResidualBlock(nn.Module):
    """Skip connections — vanishing gradient solve karta hai"""
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn1   = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn2   = nn.BatchNorm2d(channels)

    def forward(self, x):
        identity = x                    # Skip connection: original input save karo
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = out + identity            # ADD original input back!
        out = F.relu(out)
        return out

Python — PyTorch Lightning Quickstart
import pytorch_lightning as pl
import torch
import torch.nn as nn
from torch.utils.data import DataLoader

class LightningCNN(pl.LightningModule):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 32, 3)
        self.pool  = nn.MaxPool2d(2)
        self.fc    = nn.Linear(32 * 13 * 13, 10)
        self.relu  = nn.ReLU()

    def forward(self, x):
        x = self.pool(self.relu(self.conv1(x)))
        return self.fc(x.view(x.size(0), -1))

    def training_step(self, batch, batch_idx):
        x, y = batch
        logits = self(x)
        loss = nn.CrossEntropyLoss()(logits, y)
        self.log('train_loss', loss)  # Auto logging!
        return loss

    def validation_step(self, batch, batch_idx):
        x, y = batch
        logits = self(x)
        loss = nn.CrossEntropyLoss()(logits, y)
        acc = (logits.argmax(1) == y).float().mean()
        self.log('val_loss', loss)
        self.log('val_acc', acc, prog_bar=True)

    def configure_optimizers(self):
        optimizer = torch.optim.Adam(self.parameters(), lr=1e-3)
        scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50)
        return [optimizer], [scheduler]
