1.2 Vectors — Direction Wale Numbers
Vector kya hota hai?

Vector numbers ki ek ordered list hoti hai — magnitude bhi hoti hai aur direction bhi. Ise ek arrow ki tarah socho jo space mein kisi jagah point karta hai.

AI mein vectors bohot fundamental hain:

  • Word Embedding: "King" word ko [0.2, -0.5, 0.8, 0.1, ...] jaise 512-dimensional vector se represent karte hain
  • Image pixels: Ek row of pixels ek vector hai
  • User preferences: Recommendation systems mein user ek vector hota hai
  • Neural network neurons: Har layer ka output ek vector hota hai
Vector — 2D aur 3D Representation
Python — Vectors
import numpy as np

# ===== Vector Creation =====
v1 = np.array([3, 4])           # 2D vector
v2 = np.array([1, 2, 3])         # 3D vector
word_vec = np.random.randn(512)   # 512D word embedding jaise

# ===== Basic Properties =====
print("Shape:", v1.shape)        # (2,)
print("Dtype:", v1.dtype)        # int64
print("Size:", v1.size)          # 2

# ===== Vector Operations =====
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

# Addition — vectors add up component-wise
print("a + b =", a + b)           # [5, 7, 9]

# Scalar multiplication
print("2 * a =", 2 * a)           # [2, 4, 6]

# Magnitude (length) of vector
magnitude = np.linalg.norm(v1)
print(f"v1 ka magnitude = {magnitude}") # 5.0 (3-4-5 triangle)

# Unit vector (direction only, magnitude=1)
unit_v = v1 / np.linalg.norm(v1)
print("Unit vector:", unit_v)      # [0.6, 0.8]

# ===== AI Use Case: Cosine Similarity =====
# Kitna similar hain do vectors? (0 = different, 1 = same)
def cosine_similarity(v1, v2):
    dot = np.dot(v1, v2)
    mag = np.linalg.norm(v1) * np.linalg.norm(v2)
    return dot / mag

king  = np.array([0.9, 0.1, 0.8])  # "king" embedding
queen = np.array([0.8, 0.2, 0.7])  # "queen" embedding
car   = np.array([0.1, 0.9, 0.2])  # "car" embedding

print(f"King-Queen similarity: {cosine_similarity(king, queen):.3f}")
print(f"King-Car similarity:   {cosine_similarity(king, car):.3f}")

1.4 Dot Product — Similarity ka Measure
Dot Product kya hai?

Do vectors ka dot product ek single number deta hai jo batata hai ki woh kitne ek direction mein hain. Formula: a·b = Σ(aᵢ × bᵢ)

Dot Product Formula
a · b = a₁b₁ + a₂b₂ + a₃b₃ + ... + aₙbₙ
= Σᵢ aᵢ bᵢ
Agar a = [1, 2, 3] aur b = [4, 5, 6] toh:
a · b = (1×4) + (2×5) + (3×6) = 4 + 10 + 18 = 32

Dot product AI mein sabse fundamental operation hai:

  • Neural network neurons: Har neuron inputs ka dot product + bias calculate karta hai
  • Attention mechanism: Query aur Key vectors ka dot product = attention score
  • Similarity search: Vectors kitne similar hain = dot product (normalized = cosine similarity)
1.5 Matrix Multiplication — AI ka Core Operation

Matrix multiplication AI mein sabse important operation hai. Jab bhi neural network forward pass karta hai — woh matrix multiplication kar raha hota hai.

Matrix Multiplication Rule
C = A × B
A ki shape (m × k) honi chahiye, B ki (k × n) — result C ki shape (m × n) hogi.
C[i][j] = Σₖ A[i][k] × B[k][j] — Row of A dot Column of B
Matrix Multiplication — Visual Step-by-Step
Python — Matrix Multiplication
import numpy as np

A = np.array([[1, 2, 3],
               [4, 5, 6]])   # Shape: (2, 3)

B = np.array([[7,  8],
               [9,  10],
               [11, 12]])     # Shape: (3, 2)

# Method 1: np.matmul
C = np.matmul(A, B)
print("C =\n", C)              # [[58, 64], [139, 154]]
print("Shape:", C.shape)       # (2, 2)

# Method 2: @ operator (Python 3.5+)
C2 = A @ B
print("\nC2 =\n", C2)           # Same result

# ===== Neural Network Forward Pass Simulation =====
# Input: batch of 32 samples, 784 features (MNIST flattened)
X = np.random.randn(32, 784)  # (32, 784) - 32 images

# Weight matrix: 784 inputs → 256 hidden neurons
W1 = np.random.randn(784, 256) * 0.01
b1 = np.zeros(256)

# Forward pass: Hidden layer output
H = X @ W1 + b1    # (32, 784) @ (784, 256) = (32, 256)
print(f"\nHidden layer output shape: {H.shape}")

# Second layer: 256 → 10 (10 classes)
W2 = np.random.randn(256, 10) * 0.01
output = H @ W2    # (32, 256) @ (256, 10) = (32, 10)
print(f"Output shape: {output.shape}")
# Shape batata hai: 32 images, har ek ke liye 10 class scores

1.7 Eigenvalues & Eigenvectors — Data ka DNA
Eigenvalue/Eigenvector kya hai?

Ek matrix A ke liye, agar koi vector v hai aur ek number λ (lambda) hai jaise ki: Av = λv — toh v ek eigenvector hai aur λ ek eigenvalue hai. Matrix sirf vector ko scale karta hai, rotate nahi karta.

Eigenvalue Equation
A × v = λ × v
A = matrix | v = eigenvector | λ (lambda) = eigenvalue
Iska matlab: Matrix A, vector v ko sirf λ times scale karta hai — direction same rehti hai!

AI mein eigenvalues aur eigenvectors ka use:

  • PCA (Principal Component Analysis): Data ke most important directions dhundhne ke liye — dimensionality reduction
  • Google PageRank: Web pages ki ranking ke liye eigenvector use hota tha
  • Stability Analysis: Neural networks ki training stability check karne ke liye
  • Spectral Clustering: Graph-based ML algorithms mein
Python — Eigenvalues & PCA
import numpy as np
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# ===== Eigenvalue Calculation =====
A = np.array([[3, 1],
               [1, 3]])

eigenvalues, eigenvectors = np.linalg.eig(A)
print("Eigenvalues:", eigenvalues)       # [4. 2.]
print("Eigenvectors:\n", eigenvectors)   # columns are eigenvectors

# Verify: Av = λv
v = eigenvectors[:, 0]   # First eigenvector
λ = eigenvalues[0]       # First eigenvalue
print("\nAv:", A @ v)
print("λv:", λ * v)         # Same! ✓

# ===== PCA — Real AI Use Case =====
# 1000 data points with 10 features → reduce to 2
np.random.seed(42)
data = np.random.randn(1000, 10)  # 1000 samples, 10 features

pca = PCA(n_components=2)

print(f"\nOriginal shape: {data.shape}")    # (1000, 10)

print(f"Variance explained: {pca.explained_variance_ratio_}")

🎲 Probability & Statistics

Uncertainty ko measure karna — AI ke decisions ka foundation

🌟Introduction — Probability Kyun Zaroori Hai?

AI koi crystal ball nahi hai — woh certainties nahi, probabilities deal karta hai. Jab ChatGPT koi word predict karta hai, woh ek probability distribution se sample karta hai. Jab spam filter email ko spam classify karta hai, woh probability calculate karta hai. Jab recommendation system Netflix pe movie suggest karta hai, woh probability of user liking predict karta hai.

Real-World Analogy — Weather Forecast ki Tarah
"Kal 70% chance of rain" — yeh ek probability statement hai. AI hamesha aisa sochta hai. "Is email ke spam hone ka 93% chance hai." "Is patient ke diabetes hone ka 67% probability hai." "Next word 'the' hone ki 34% probability, 'a' ki 28%..." — Probability ek AI ki natural language hai!
2.1 Mean — Average Value
Mean (Arithmetic Mean) kya hai?

Saare values ka sum, total count se divide karna. Yeh central tendency ka sabse common measure hai — "typical value" batata hai.

Mean Formula
μ = (1/N) × Σᵢ xᵢ = (x₁ + x₂ + ... + xₙ) / N
μ (mu) = population mean | N = total count | xᵢ = individual values
Example: [2, 4, 6, 8, 10] → Mean = (2+4+6+8+10)/5 = 30/5 = 6

AI mein mean kahan use hota hai:

  • Batch Normalization: Neural networks mein har layer ke outputs ka mean subtract karte hain
  • Mean Squared Error (MSE): Most common loss function
  • Feature scaling: Data preprocessing mein mean use karte hain
  • Ensemble methods: Multiple models ke predictions ka mean = final prediction
2.2 Median — Middle Value
Median kya hai?

Data ko sort karne ke baad beech wali value. Yeh outliers se affected nahi hoti — isliye median mean se zyada "robust" hoti hai.

Analogy — Salary Example
Ek company mein 9 employees hain jिनकी salary hai: [20K, 25K, 30K, 32K, 35K, 40K, 45K, 50K, 500K].
Mean = ~86K (CEO ki 500K salary se distorted!) | Median = 35K (actually representative!)
Isliye median zyada "fair" hota hai jab outliers ho.
2.3 Mode — Sabse Common Value
Mode kya hai?

Jo value sabse zyada baar repeat hoti hai. Categorical data ke liye best central tendency measure. Ek dataset bimodal ya multimodal bhi ho sakta hai.

2.4 Variance — Spread of Data
Variance kya hai?

Data kitna spread out hai mean se — yeh variability ka measure hai. High variance = data bohot scattered. Low variance = data mean ke paas.

Variance Formula
σ² = (1/N) Σᵢ (xᵢ − μ)²
σ² (sigma squared) = variance | μ = mean | N = count
Har value ko mean se subtract karo, square karo, phir sab ka average lo.
Squaring kyun? Negative differences positive ho jaate hain aur bade differences zyada penalize hote hain.
2.5 Standard Deviation — Variance ka Square Root
Standard Deviation
σ = √σ² = √[(1/N) Σᵢ (xᵢ − μ)²]
σ (sigma) = standard deviation | Same unit mein hoti hai data ke
Variance: squared units mein (e.g., meters²) | Std Dev: same units (meters)
Isliye std dev zyada interpretable hai!
Mean, Variance, Std Dev — Visual Comparison
Python — Statistics Fundamentals
import numpy as np
from scipy import stats

# ===== Sample Dataset: AI model evaluation scores =====
scores = np.array([72, 85, 78, 92, 68, 88, 75, 95, 71, 83])

# ===== Measures of Central Tendency =====
mean   = np.mean(scores)
median = np.median(scores)
mode   = stats.mode(scores)

print(f"Mean   (Average):   {mean:.2f}")     # 80.70
print(f"Median (Middle):    {median:.2f}")   # 81.50
print(f"Mode   (Frequent):  {mode.mode[0]}") # No repeat = each appears once

# ===== Measures of Spread =====
variance = np.var(scores)
std_dev  = np.std(scores)
data_range = np.max(scores) - np.min(scores)

print(f"\nVariance (σ²):      {variance:.2f}")
print(f"Std Dev  (σ):       {std_dev:.2f}")
print(f"Range:              {data_range}")
print(f"Min: {np.min(scores)}, Max: {np.max(scores)}")

# ===== AI Context: Evaluating Model Performance =====
model_a_scores = np.array([80, 81, 79, 82, 80])  # Consistent model
model_b_scores = np.array([90, 60, 85, 55, 95])  # Inconsistent model

print(f"\nModel A — Mean: {np.mean(model_a_scores):.1f}, Std: {np.std(model_a_scores):.1f}")
print(f"Model B — Mean: {np.mean(model_b_scores):.1f}, Std: {np.std(model_b_scores):.1f}")
# Model A low std dev → reliable! Model B high std dev → unreliable!

# ===== Z-Score: Standardization =====
# AI mein input features normalize karte hain
z_scores = (scores - np.mean(scores)) / np.std(scores)
print(f"\nZ-scores: {z_scores.round(2)}")
# Positive = above mean, Negative = below mean

2.7 Conditional Probability — "Given That" ki Probability
Conditional Probability kya hai?

P(A|B) = "B hone ki condition mein A ki probability". Event B already ho chuka hai — iske baad A kitna likely hai? Yeh AI mein bayesian thinking ka base hai.

Conditional Probability Formula
P(A|B) = P(A ∩ B) / P(B)
P(A|B) = A ki probability given B | P(A ∩ B) = dono saath hone ki probability
Example: P(Spam | "Free Money") = P(Spam AND "Free Money") / P("Free Money")
Agar "Free Money" wali 90% emails spam hain → P(Spam|"Free Money") = 0.90
2.8 Bayes' Theorem — AI ka Core Foundation
Bayes' Theorem kya hai?

Evidence ke basis pe beliefs update karna. Yeh Naive Bayes classifier, Bayesian neural networks, spam filters, aur kaafi AI algorithms ka foundation hai.

Bayes' Theorem
P(A|B) = [P(B|A) × P(A)] / P(B)
Posterior: P(A|B) — Evidence B dekhne ke baad A ki updated probability
Likelihood: P(B|A) — Agar A true hota toh B ka kitna chance tha
Prior: P(A) — Evidence se pehle A ki probability
Evidence: P(B) — B ki overall probability (normalization constant)
Step-by-Step Bayes Example — Medical Diagnosis
Situation: Ek disease 1% population mein hoti hai. Test 95% accurate hai.

Tum test positive aaye. Kya tum actually beemar ho?

• P(Disease) = 0.01 (1% population mein hoti hai) ← Prior
• P(Positive | Disease) = 0.95 (95% accurate test) ← Likelihood
• P(Positive | No Disease) = 0.05 (5% false positive)
• P(Positive) = (0.95×0.01) + (0.05×0.99) = 0.0095 + 0.0495 = 0.059

P(Disease | Positive) = (0.95 × 0.01) / 0.059 ≈ 0.161 = 16.1%!

Surprise! Even with 95% accurate test, agar disease rare hai toh positive test bhi sirf 16% chance deta hai! Yahi Bayes ka power hai — prior probability matter karta hai!
Python — Bayes' Theorem & Naive Bayes
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer
import numpy as np

# ===== Manual Bayes Calculation =====
# Email spam detection example

# Prior probabilities
P_spam    = 0.30   # 30% emails spam hain
P_not_spam = 0.70  # 70% emails legitimate hain

# Likelihood: "FREE" word ka probability
P_free_given_spam    = 0.80  # Spam mein "FREE" 80% baar
P_free_given_not_spam = 0.05 # Normal email mein "FREE" 5%

# Evidence: P("FREE")
P_free = (P_free_given_spam * P_spam +
           P_free_given_not_spam * P_not_spam)

# Posterior: P(Spam | "FREE") = Bayes Theorem
P_spam_given_free = (P_free_given_spam * P_spam) / P_free

print(f"P(FREE): {P_free:.3f}")
print(f"P(Spam | FREE word): {P_spam_given_free:.3f} = {P_spam_given_free*100:.1f}%")

# ===== Naive Bayes Spam Classifier =====
# Training data
emails = [
    "free money click here win prize",      # spam
    "dear friend meeting tomorrow office",   # ham
    "win lottery free cash urgent",           # spam
    "project deadline next week team",        # ham
    "click free offer limited time prize",   # spam
    "quarterly report attached review",      # ham
]
labels = [1, 0, 1, 0, 1, 0]  # 1=spam, 0=ham

# Vectorize text
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(emails)

# Train Naive Bayes (internally uses Bayes Theorem!)
clf = MultinomialNB()
clf.fit(X, labels)

# Predict
test_emails = ["free prize click win money", "team meeting agenda next week"]
X_test = vectorizer.transform(test_emails)
predictions = clf.predict(X_test)
probabilities = clf.predict_proba(X_test)

for email, pred, prob in zip(test_emails, predictions, probabilities):
    label = "SPAM" if pred == 1 else "HAM"
    print(f"Email: '{email[:30]}...'")
    print(f"  Prediction: {label} | Spam prob: {prob[1]:.2%}\n")

2.9 Hypothesis Testing — Statistical Decision Making
Hypothesis Testing kya hai?

Data ke basis pe statistical decisions lena. Kya ek AI model dusre se genuinely better hai, ya sirf luck? Hypothesis testing batata hai. Null Hypothesis (H₀) = "koi difference nahi" aur Alternative Hypothesis (H₁) = "difference hai".

ConceptAI ContextMeaning
p-valueA/B testing models≤0.05 = statistically significant
Null HypothesisModel A == Model BNo difference claim
Type I ErrorFalse PositiveH₀ reject kiya but true tha
Type II ErrorFalse NegativeH₀ accept kiya but false tha
Confidence IntervalModel accuracy range95% CI: [0.82, 0.89]
Statistical PowerTest sensitivityTrue effect detect karne ki ability
Python — A/B Testing for AI Models
from scipy import stats
import numpy as np

# ===== A/B Test: Model A vs Model B =====
# Model accuracy scores on 30 test batches
np.random.seed(42)
model_a = np.random.normal(loc=0.85, scale=0.05, size=30)
model_b = np.random.normal(loc=0.88, scale=0.05, size=30)

print(f"Model A — Mean: {model_a.mean():.4f}, Std: {model_a.std():.4f}")
print(f"Model B — Mean: {model_b.mean():.4f}, Std: {model_b.std():.4f}")

# T-test: Kya difference significant hai?
t_stat, p_value = stats.ttest_ind(model_a, model_b)

print(f"\nT-statistic: {t_stat:.4f}")
print(f"P-value:     {p_value:.4f}")

if p_value < 0.05:
    print("✅ Statistically significant! Model B genuinely better hai.")
else:
    print("❌ Not significant. Difference luck ho sakta hai.")

# ===== Confidence Interval =====
ci = stats.t.interval(
    confidence=0.95,
    df=len(model_b) - 1,
    loc=np.mean(model_b),
    scale=stats.sem(model_b)
)
print(f"\nModel B 95% CI: [{ci[0]:.4f}, {ci[1]:.4f}]")
# "95% sure hai ki Model B ka true accuracy is range mein hai"

Neural network training ka engine — gradients aur optimization

🌟Introduction — Calculus Kyun?

Neural network training ka poora process ek word mein describe ho sakta hai: optimization. Hum chahte hain ki model ka loss (error) minimize ho. Lekin kitne millions of parameters hain! Inhe kaise adjust karein? Answer hai: Gradient Descent, aur iske liye hume derivatives aur gradients samajhne padenge.

Analogy — Pahad Se Utrna (Blind Descent)
Socho tum ek pahaad ke upar ho, andhera hai, kuch dikhai nahi deta. Tumhe valley (lowest point) tak pahunchna hai — jo loss = minimum hai. Tum kya karoge?

Har step pe apne paon se mahsoos karo — kaunsi side steep hai (gradient). Phir usi direction mein ek chota step lo. Repeat. Yahi hai Gradient Descent! Derivative batata hai — "is jagah pe function kitni tezi se change ho raha hai" — aur hum usi information use karke weights update karte hain.
3.1 Derivatives — Instantaneous Rate of Change
Derivative kya hai?

Derivative batata hai ki ek function kitni tezi se change ho raha hai ek particular point pe. Isko slope bhi keh sakte hain — curve ke kisi bhi point pe tangent line ka slope.

Derivative — Basic Formula
f'(x) = df/dx = lim(h→0) [f(x+h) - f(x)] / h
Common derivatives:
d/dx (xⁿ) = n·xⁿ⁻¹  |  d/dx (eˣ) = eˣ  |  d/dx (ln x) = 1/x
d/dx (sin x) = cos x  |  d/dx (constant) = 0
Derivative — Slope at Different Points
💡 Beginner Intuition
Derivative basically ek question ka answer deta hai: "Agar main x mein thoda sa badhaunga, toh y mein kitna badhega?"
• Derivative positive → x badhane se y bhi badhega
• Derivative negative → x badhane se y ghattega
• Derivative zero → hum minimum ya maximum pe hain!
Neural network mein: derivative bolta hai "is weight ko badhane se loss badhega ya ghattega" — isliye hum opposite direction mein jaate hain.
3.2 Gradients — Multi-Variable Derivative
Gradient kya hai?

Jab function mein ek se zyada variables hain (jaise neural network ke lakho weights), toh gradient ek vector hota hai jo har variable ke partial derivative ko hold karta hai. Gradient batata hai — "highest increase ki direction kya hai".

Gradient of f(x, y)
∇f = [∂f/∂x, ∂f/∂y]
∇ (nabla) = gradient | ∂f/∂x = partial derivative w.r.t x (y fixed)
Gradient Vector points in direction of steepest ascent
Gradient Descent: Gradient ki opposite direction mein move karo (minimize karne ke liye)

w = w - α × ∇L(w)   (weight update rule)
3.3 Partial Derivatives — Ek Time pe Ek Variable
Partial Derivative kya hai?

Jab multiple variables hain, toh ek ek karke derivative lete hain — baaki sab ko constant maanke. Neural network mein har weight ke liye alag partial derivative calculate hoti hai.

Partial Derivative Example
f(x, y) = 3x² + 2xy + y³
∂f/∂x = 6x + 2y    (y ko constant maano)
∂f/∂y = 2x + 3y²    (x ko constant maano)
3.4 Chain Rule — Backpropagation ka Heart
Chain Rule kya hai?

Jab functions ek doosre ke andar hoon (composite functions), toh derivative nikalne ka rule. Neural networks mein Backpropagation poori tarah chain rule use karta hai — output se input tak gradients compute karne ke liye.

Chain Rule
dy/dx = (dy/du) × (du/dx)
Agar y = f(u) aur u = g(x), toh dy/dx = f'(u) × g'(x)

Neural Network example:
Loss = L(output), output = σ(Wx + b)
∂L/∂W = (∂L/∂output) × (∂output/∂W) ← Chain Rule!
Backpropagation — Chain Rule in Neural Networks
3.5 Gradient Descent — Optimization ka Algorithm
Gradient Descent kya hai?

Loss function ko minimize karne ka iterative algorithm. Har iteration mein weights ko gradient ki opposite direction mein update karo — "downhill" jaao loss landscape mein.

Weight Update Rule
w_new = w_old − α × ∂L/∂w
w = weight parameter | α (alpha) = learning rate (step size)
∂L/∂w = gradient of loss w.r.t weight
Minus sign kyun? Gradient steep ascent direction hai — hum descent chahte hain!
TypeBatch SizeProsConsUse Case
Batch GDFull datasetStable, accurate gradientSlow, memory hungrySmall datasets
Stochastic GD1 sampleFast, noisy updates help escape local minimaNoisy, unstableOnline learning
Mini-batch GD32-512Balance of speed & stabilityHyperparameter tuningMost practical (default)
AdamMini-batchAdaptive, fast convergenceMemory overheadAlmost always! (default)
Python — Gradient Descent from Scratch
import numpy as np
import matplotlib.pyplot as plt

# ===== Manual Gradient Descent =====
# Problem: f(x) = x^2 + 4x + 4 ko minimize karo
# f'(x) = 2x + 4 (derivative)
# Minimum at x = -2 (f'(x) = 0 → 2x + 4 = 0 → x = -2)

def f(x):
    return x**2 + 4*x + 4

def gradient_f(x):
    return 2*x + 4   # Derivative of f(x)

# Gradient Descent
x = 10.0              # Starting point (random)
learning_rate = 0.1   # Step size
history = [x]

print("Gradient Descent Steps:")
print(f"{'Step':>4} | {'x':>8} | {'f(x)':>10} | {'gradient':>10}")
print("-" * 40)

for step in range(20):
    grad = gradient_f(x)         # Gradient calculate karo
    x = x - learning_rate * grad  # Weight update
    history.append(x)
    if step % 4 == 0:
        print(f"{step:>4} | {x:>8.4f} | {f(x):>10.4f} | {grad:>10.4f}")

print(f"\n🎯 Final x = {x:.4f} (expected: -2.0)")
print(f"🎯 Final f(x) = {f(x):.6f} (expected: 0.0)")

# ===== PyTorch Autograd (Real AI use) =====
import torch

# PyTorch automatically computes gradients!
x = torch.tensor(10.0, requires_grad=True)  # Track gradient

for i in range(20):
    loss = x**2 + 4*x + 4   # Forward pass
    loss.backward()           # Autograd: compute gradients
    with torch.no_grad():
        x -= 0.1 * x.grad      # Update
        x.grad.zero_()          # Zero gradient for next iteration

print(f"\nPyTorch result: x = {x.item():.4f}")

Python — Activation Functions & Their Derivatives
import numpy as np

# ===== Activation Functions =====
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    s = sigmoid(x)
    return s * (1 - s)   # Backprop mein use hota hai

def relu(x):
    return np.maximum(0, x)

def relu_derivative(x):
    return (x > 0).astype(float)  # 1 if positive, 0 if negative

def softmax(x):
    e_x = np.exp(x - np.max(x))  # Numerical stability
    return e_x / e_x.sum()

# Test values
z = np.array([-2, -1, 0, 1, 2], dtype=float)

print("z        :", z)
print("sigmoid  :", sigmoid(z).round(3))
print("sig'(z)  :", sigmoid_derivative(z).round(3))
print("relu     :", relu(z))
print("relu'(z) :", relu_derivative(z))

# LLM output — next token probabilities
logits = np.array([2.5, 1.2, 0.8, 3.1, 0.4])  # Raw scores
probs = softmax(logits)
print(f"\nLogits:       {logits}")
print(f"Probabilities: {probs.round(3)}")
print(f"Sum: {probs.sum():.1f}")  # Always 1.0
print(f"Predicted token: {np.argmax(probs)}")  # Highest prob token

⚠️ Common Mistakes
Vanishing Gradient: Deep networks mein sigmoid/tanh use karne se gradients bahut chhote ho jaate hain back layers tak — training ruk jaati hai. Solution: ReLU/GELU use karo, batch normalization karo.

Exploding Gradient: Gradients bohot bade ho jaate hain — NaN/Inf values. Solution: Gradient clipping use karo.

Learning Rate Too High: Loss oscillate karta hai, converge nahi hota.
Learning Rate Too Low: Training bohot slow — local minima mein phase jaata hai.
