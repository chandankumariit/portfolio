🤖 1.1 — Multi-Agent System Kya Hota Hai?

Simple Definition: Jab multiple AI agents milke ek common goal achieve karte hain, toh us poore system ko Multi-Agent System (MAS) kehte hain.

Intuition: Socho ek badi company jahan alag-alag departments hote hain — HR, Finance, Engineering, Marketing. Koi ek department sabkuch akele nahi kar sakta. Sab milke kaam karte hain. Exactly waise hi MAS mein multiple AI agents hote hain, har ek apna kaam karta hai, sab milke bada goal complete karte hain.

📖 Technical Definition

A Multi-Agent System ek aisa computational environment hai jismein multiple autonomous agents exist karte hain jo environment ko perceive karte hain, actions lete hain, aur dusre agents ke saath communicate karte hain apne individual ya shared goals achieve karne ke liye.

Multi-Agent Systems Kyun Chahiye? (Why MAS?)

  1. 1

    Task Decomposition

    Ek bada task akela agent nahi kar sakta. MAS mein task chhote pieces mein todte hain aur parallel mein solve karte hain. Jaise ek company mein departments.

  2. 2

    Specialization

    Har agent ek specific skill mein expert hota hai. Research agent sirf research karta hai, coder sirf code karta hai. Generalist agent se better results aate hain.

  3. 3

    Fault Tolerance

    Agar ek agent fail ho jaye, poora system crash nahi hota. Dusre agents kaam continue karte hain ya failed agent ka kaam le lete hain.

  4. 4

    Scalability

    Zyada kaam aaye toh zyada agents add karo. Simple horizontal scaling. Single agent mein ye possible nahi hota.

  5. 5

    Complex Problem Solving

    Real-world problems bohot complex hote hain. Multiple agents milke multi-dimensional problems solve kar sakte hain jo single agent nahi kar sakta.

🌍 Real-World Analogy — Software Development Company

Ek startup mein:

  • Product Manager Agent — Requirements gather karta hai, user stories likhta hai
  • Architect Agent — System design karta hai, technology choose karta hai
  • Developer Agent — Code likhta hai, features implement karta hai
  • QA Agent — Testing karta hai, bugs dhundta hai
  • DevOps Agent — Deploy karta hai, monitoring karta hai

Sab milke ek complete software product deliver karte hain! Yahi Multi-Agent System hai.

🤝 1.2 — Collaborative Agents

Kya hai? Jab multiple agents ek dusre ke saath cooperate karte hain, information share karte hain, aur ek common goal achieve karne ke liye milke kaam karte hain — unhe Collaborative Agents kehte hain.

Intuition: Jaise football team mein 11 players — sab alag positions pe hote hain (goalkeeper, defenders, forwards) lekin ek hi goal hai: match jeetnaa. Koi akele kaam nahi karta, sab coordinate karte hain.

💡 Key Insight

Collaborative agents mein competition nahi hoti — sab ek hi direction mein kaam karte hain. Har agent dusre agent ki success chahta hai kyunki isse overall system succeed karta hai.

Collaborative Agents ke Properties:

🎯

Shared Goal

Sab agents ka ek common objective hota hai. Individual goals bhi ho sakte hain but overall goal shared hoti hai.

💬

Open Communication

Agents freely information share karte hain. Koi data hide nahi karta. Transparency hoti hai system mein.

⚡

Mutual Benefit

Ek agent ka success dusre ko bhi benefit karta hai. Win-win situation hoti hai sab ke liye.

🔄

Task Division

Bada kaam chhote subtasks mein divide hota hai. Har agent apna subtask complete karta hai.

from dataclasses import dataclass
from typing import List, Dict
import time

@dataclass
class SharedContext:
    """Shared memory — sab agents yahan se padhte aur likhte hain"""
    topic: str = ""
    research_data: List[str] = None
    outline: List[str] = None
    final_blog: str = ""
    status: Dict[str, str] = None

    def __post_init__(self):
        self.research_data = []
        self.outline = []
        self.status = {}

class ResearchAgent:
    """Agent 1: Sirf research karta hai"""
    def __init__(self, name="ResearchBot"):
        self.name = name

    def run(self, ctx: SharedContext):
        print(f"🔍 {self.name}: Research shuru karta hoon '{ctx.topic}' pe...")
        time.sleep(0.5)  # Simulate API call

        # Research results shared context mein daalo
        ctx.research_data = [
            f"{ctx.topic} ke 5 main benefits hain",
            f"{ctx.topic} industry mein 2024 mein bohat use ho raha hai",
            f"{ctx.topic} beginner ke liye best practices",
            f"Case study: Company X ne {ctx.topic} se 40% efficiency badhaya"
        ]
        ctx.status[self.name] = "DONE"
        print(f"✅ {self.name}: Research complete! {len(ctx.research_data)} points mila")

class OutlineAgent:
    """Agent 2: Research padh ke outline banata hai"""
    def __init__(self, name="OutlineBot"):
        self.name = name

    def run(self, ctx: SharedContext):
        # Wait karo jab tak research complete na ho
        while "ResearchBot" not in ctx.status:
            time.sleep(0.1)

        print(f"📋 {self.name}: Research padha, ab outline bana raha hoon...")
        time.sleep(0.3)

        ctx.outline = [
            "1. Introduction — क्या है?",
            "2. Main Benefits — kyun important hai",
            "3. Step-by-step Guide — kaise use karein",
            "4. Real Case Study — industry example",
            "5. Conclusion aur Next Steps"
        ]
        ctx.status[self.name] = "DONE"
        print(f"✅ {self.name}: Outline ready with {len(ctx.outline)} sections")

class WriterAgent:
    """Agent 3: Outline aur research se final blog likhta hai"""
    def __init__(self, name="WriterBot"):
        self.name = name

    def run(self, ctx: SharedContext):
        # Dono agents complete hone ka wait karo
        while "OutlineBot" not in ctx.status:
            time.sleep(0.1)

        print(f"✍️ {self.name}: Ab final blog likh raha hoon...")
        time.sleep(0.5)

        ctx.final_blog = f"""
=== BLOG: {ctx.topic.upper()} ===

{chr(10).join(ctx.outline)}

[Research Data Used: {len(ctx.research_data)} points]
[Status: COMPLETE ✅]
"""
        ctx.status[self.name] = "DONE"
        print(f"✅ {self.name}: Blog likh diya!")

# === Main: Sab agents milke kaam karte hain ===
def run_collaborative_system(topic: str):
    ctx = SharedContext(topic=topic)

    # Agents banao
    researcher = ResearchAgent()
    outliner   = OutlineAgent()
    writer     = WriterAgent()

    # Sequential run (real mein threading hoti)
    researcher.run(ctx)
    outliner.run(ctx)
    writer.run(ctx)

    print("\n🎉 FINAL OUTPUT:")
    print(ctx.final_blog)
    return ctx

result = run_collaborative_system("Multi-Agent AI Systems")

⚔️ 1.3 — Competitive Agents

Kya hai? Jab multiple agents same resources ke liye compete karte hain, ya unke goals ek dusre se oppose karte hain — unhe Competitive Agents kehte hain.

Intuition: Auction mein buyers — sab ek hi cheez kharidna chahte hain, aur sabse zyada bid karne waala jeetta hai. Ya chess khelne mein — dono players ek dusre ko harana chahte hain.

⚠️ Important Note

Competitive agents ka matlab "kharab" nahi hai! Market simulations, game theory, adversarial testing aur auction systems mein competition zaruri hoti hai for optimal outcomes.

Collaborative vs Competitive — Side by Side:

🤝 Collaborative

  • Ek common goal
  • Information freely share
  • Sab win karte hain
  • Teamwork important
  • Example: Research + Writer + QA team
  • Use: Product development, complex workflows

⚔️ Competitive

  • Opposing goals
  • Information hide karte hain
  • Zero-sum ya winner-take-all
  • Self-interest important
  • Example: Auction bidder agents
  • Use: Market simulations, game AI, adversarial testing

# Competitive Agents — Auction Simulation
# Multiple bidder agents ek item ke liye compete karte hain

import random

class BidderAgent:
    """Ek bidder agent — apna budget secret rakhta hai"""
    def __init__(self, name: str, budget: float):
        self.name = name
        self.budget = budget  # SECRET — dusre agents ko pata nahi
        self.current_bid = 0

    def make_bid(self, current_price: float, item_value: float) -> float:
        """Strategy: Apni value se kam bid karo, budget se zyada nahi"""
        if current_price >= self.budget:
            print(f"  ❌ {self.name}: Budget khatam! ({self.budget:.0f}). Out.")
            return 0  # Bahaar ho gaya

        # Random strategy — value se 5-15% kam bid karo
        discount = random.uniform(0.05, 0.15)
        max_bid = min(self.budget, item_value * (1 - discount))
        bid = current_price + random.uniform(10, 50)
        bid = min(bid, max_bid)

        if bid <= current_price:
            print(f"  🤐 {self.name}: Bid nahi karunga (current price too high)")
            return 0

        self.current_bid = bid
        print(f"  💰 {self.name}: Bid karta hoon ₹{bid:.0f}")
        return bid

class AuctionSystem:
    """Auction orchestrator — fair play ensure karta hai"""
    def __init__(self, item_name: str, starting_price: float):
        self.item = item_name
        self.price = starting_price
        self.winner = None

    def run_auction(self, bidders: list):
        print(f"\n🔨 AUCTION: '{self.item}' ke liye bidding shuru!")
        print(f"💵 Starting Price: ₹{self.price:.0f}\n")

        round_num = 1
        while True:
            print(f"--- Round {round_num} --- Current: ₹{self.price:.0f} ---")
            best_bid = 0
            best_bidder = None

            for bidder in bidders:
                bid = bidder.make_bid(self.price, 5000)
                if bid > best_bid:
                    best_bid = bid
                    best_bidder = bidder

            if best_bid == 0 or best_bidder is None:
                print("\n🎯 Koi aur bid nahi! Auction khatam!")
                break

            self.price = best_bid
            self.winner = best_bidder
            print(f"  🏆 Current winner: {best_bidder.name} @ ₹{best_bid:.0f}\n")
            round_num += 1

            if round_num > 5: break  # Max rounds

        print(f"🎉 WINNER: {self.winner.name} ne ₹{self.price:.0f} mein '{self.item}' jeeta!")

# Agents banao — har ek ka secret budget
agents = [
    BidderAgent("RahulBot", budget=3000),
    BidderAgent("PriyaBot", budget=4500),
    BidderAgent("ArjunBot", budget=3800),
]

🌐 1.4 — Distributed Systems Basics

Kya hai? Distributed System ek aisa system hai jismein multiple computers/processes network pe kaam karte hain, lekin user ko ek hi system lagta hai. Multi-Agent Systems mein agents alag machines ya processes pe run kar sakte hain.

Intuition: WhatsApp ka server — duniya mein kai jagah hain (US, Europe, India), lekin tumhe ek unified app milti hai. Sab computers milke ek system ki tarah behave karte hain.

💬 2.1 — Message Passing

Kya hai? Message passing mein Agent A seedha Agent B ko ek message bhejta hai. Jaise WhatsApp pe message karna. Koi middleman nahi, direct communication.

Types of Message Passing:

⚡

Synchronous (Sync)

Sender wait karta hai reply ke liye. Blocking call. Jaise phone call — dono ek saath online hote hain.

📮

Asynchronous (Async)

Sender message bhejta hai aur apna kaam karta rehta hai. Jaise email — bhejne ke baad reply ka wait nahi. Queue mein jata hai.

📡

Broadcast

Ek agent sab agents ko message bhejta hai ek saath. "Important announcement" type. Like radio broadcast.

🎯

Multicast

Selected group of agents ko message. "Team A" ko bhejo. Broadcast se targeted, point-to-point se broader.

PYTHON — Message Passing System 
# Message Passing — Agents ke beech communication
# Queue-based async message passing

from queue import Queue
from dataclasses import dataclass, field
from typing import Any, Optional
from datetime import datetime

@dataclass
class Message:
    """Standard message format agents ke beech"""
    sender: str          # Kaun bhej raha hai
    receiver: str        # Kisko bhej raha hai
    msg_type: str        # "request", "response", "notification"
    content: Any         # Actual data
    timestamp: str = field(default_factory=lambda: datetime.now().strftime("%H:%M:%S"))
    priority: int = 5   # 1=highest, 10=lowest

class MessageBus:
    """Central message router — jaise post office"""
    def __init__(self):
        self.queues: dict = {}     # Agent name → Queue
        self.message_log = []      # Audit trail

    def register_agent(self, agent_name: str):
        """Naya agent register karo — inbox banao"""
        self.queues[agent_name] = Queue()
        print(f"📬 Agent '{agent_name}' registered with message bus")

    def send(self, message: Message):
        """Message bhejo"""
        if message.receiver not in self.queues:
            raise ValueError(f"Agent '{message.receiver}' registered nahi hai!")

        self.queues[message.receiver].put(message)
        self.message_log.append(message)
        print(f"  [{message.timestamp}] 📨 {message.sender} → {message.receiver}: [{message.msg_type}]")

    def receive(self, agent_name: str, block=False) -> Optional[Message]:
        """Message receive karo (non-blocking by default)"""
        if self.queues[agent_name].empty():
            return None
        return self.queues[agent_name].get(block=block)

    def broadcast(self, sender: str, msg_type: str, content: Any):
        """Sab agents ko message bhejo"""
        print(f"  📢 BROADCAST from {sender}:")
        for agent in self.queues:
            if agent != sender:  # Khud ko message mat bhejo
                msg = Message(sender=sender, receiver=agent,
                               msg_type=msg_type, content=content)
                self.send(msg)

# === Usage Example ===
bus = MessageBus()

# Agents register karo
for agent in ["orchestrator", "researcher", "writer", "reviewer"]:
    bus.register_agent(agent)

print("\n--- Message Passing Demo ---")

# Orchestrator sab agents ko task assign karta hai (broadcast)
bus.broadcast(
    sender="orchestrator",
    msg_type="task_assign",
    content={"task": "Write blog about Multi-Agent Systems", "deadline": "2hr"}
)

# Researcher orchestrator ko result bhejta hai (direct message)
bus.send(Message(
    sender="researcher",
    receiver="orchestrator",
    msg_type="task_complete",
    content={"status": "done", "data": "10 research points found"}
))

# Messages process karo
print("\n--- Orchestrator ki inbox check karo ---")
while True:
    msg = bus.receive("orchestrator")
    if not msg: break
    print(f"  ✉️ Received from {msg.sender}: {msg.content}")

🗄️ 2.2 — Shared Memory

Kya hai? Shared Memory mein ek common storage hota hai jahan sab agents data read aur write karte hain. Koi direct messaging nahi — sab ek jagah se information lete hain.

Intuition: Office ka whiteboard — koi bhi kuch likh sakta hai, koi bhi padh sakta hai. "Task List" pe likha hai kya karna hai, sab dekhte hain aur apna kaam update karte hain.

⚠️ Race Condition — Common Problem

Jab do agents ek saath same memory location pe write karte hain, toh Race Condition ho sakti hai. Ek agent dusre ka data overwrite kar deta hai. Iske liye locks aur mutex use karte hain.

PYTHON — Shared Memory with Thread Safety 
import threading
from typing import Any, Dict, List

class SharedMemory:
    """
    Thread-safe Shared Memory — sab agents yahan se padhte/likhte hain
    Race conditions se bachne ke liye threading.Lock use karte hain
    """
    def __init__(self):
        self._store: Dict[str, Any] = {}
        self._lock = threading.Lock()  # Race condition se bachao
        self._history: List[dict] = []

    def write(self, key: str, value: Any, agent_name: str):
        """Thread-safe write operation"""
        with self._lock:  # Lock lao — sirf main likh sakta hoon
            old_val = self._store.get(key)
            self._store[key] = value
            self._history.append({
                "agent": agent_name,
                "action": "write",
                "key": key,
                "old_value": old_val,
                "new_value": value
            })
            print(f"  ✏️ [{agent_name}] wrote '{key}' = {str(value)[:50]}")

    def read(self, key: str, default=None) -> Any:
        """Thread-safe read operation"""
        with self._lock:  # Lock — consistent read
            return self._store.get(key, default)

    def update_list(self, key: str, value: Any, agent_name: str):
        """Atomically list mein append karo"""
        with self._lock:
            if key not in self._store:
                self._store[key] = []
            self._store[key].append(value)
            print(f"  ➕ [{agent_name}] appended to '{key}'")

    def get_snapshot(self) -> dict:
        """Current state ka snapshot lo"""
        with self._lock:
            return dict(self._store)

    research_data = ["Fact 1", "Fact 2", "Fact 3"]
    memory.write("research_data", research_data, "ResearchAgent")
    memory.write("research_status", "done", "ResearchAgent")

def writer_agent_work():

    data = memory.read("research_data", [])
    blog = f"Blog based on {len(data)} research points"
    memory.write("final_blog", blog, "WriterAgent")

# Threads mein run karo

t2 = threading.Thread(target=writer_agent_work)

print("\n--- Shared Memory Demo ---")
t1.start()
t1.join()   # Research complete hone do pehle
t2.start()
t2.join()

for k, v in snapshot.items():

⚡ 2.3 — Event-Driven Systems

Kya hai? Event-driven system mein agents events publish karte hain aur dusre agents jo interested hain woh subscribe karte hain. Jab koi event hota hai, subscribers ko automatically notification jaata hai.

Intuition: YouTube Subscribe button — jab koi creator naya video dalti hai (event), sab subscribers ko notification milti hai. Creator ko individually message nahi bhejne padte.

📢 Publisher Agent (Event creator)
🔔 Event Bus (Middleman)
👂 Subscriber Agents (Listeners)
PYTHON — Event-Driven Multi-Agent System 
# Event-Driven Architecture — Pub/Sub Pattern
# Agents events publish karte hain, dusre subscribe karte hain

from typing import Callable, Dict, List, Any
from dataclasses import dataclass
import time

@dataclass
class Event:
    event_type: str    # "task_complete", "error", "data_ready"
    source: str        # Kaun agent ne event fire kiya
    data: Any          # Event ka payload

class EventBus:
    """
    Central Event Bus — Publish/Subscribe pattern
    Agents loosely coupled rahte hain — direct dependency nahi
    """
    def __init__(self):
        # event_type → list of handler functions
        self._subscribers: Dict[str, List[Callable]] = {}
        self._event_log: List[Event] = []

    def subscribe(self, event_type: str, handler: Callable, agent_name: str):
        """Kisi event ko sunna shuru karo"""
        if event_type not in self._subscribers:
            self._subscribers[event_type] = []
        self._subscribers[event_type].append(handler)
        print(f"  🔔 {agent_name} subscribed to '{event_type}'")

    def publish(self, event: Event):
        """Event fire karo — sab subscribers ko notify karo"""
        self._event_log.append(event)
        print(f"\n  🔥 EVENT FIRED: [{event.event_type}] by {event.source}")

        handlers = self._subscribers.get(event.event_type, [])
        if not handlers:
            print(f"  ⚠️ Koi subscriber nahi hai '{event.event_type}' ke liye")
            return

        for handler in handlers:
            print(f"  ↳ Handling...")
            handler(event)  # Har subscriber ka handler call karo

# === Agents Define Karo ===

class DataIngestionAgent:
    def __init__(self, bus: EventBus):
        self.bus = bus

    def fetch_data(self):
        print("📥 DataIngestion: Data fetch kar raha hoon...")
        time.sleep(0.5)
        data = {"rows": 1500, "source": "database"}

        # Event publish karo jab data ready ho
        self.bus.publish(Event(
            event_type="data_ready",
            source="DataIngestionAgent",
            data=data
        ))

class ProcessingAgent:
    """data_ready event pe kaam karta hai"""
    def __init__(self, bus: EventBus):
        self.bus = bus
        # Subscribe to data_ready events
        bus.subscribe("data_ready", self.on_data_ready, "ProcessingAgent")

    def on_data_ready(self, event: Event):
        print(f"  ⚙️ ProcessingAgent: {event.data['rows']} rows process kar raha hoon")
        result = {"processed": event.data["rows"], "insights": 42}
        self.bus.publish(Event("processing_done", "ProcessingAgent", result))

    """Sab complete events pe notification bhejta hai"""
    def __init__(self, bus: EventBus):

    def on_data_ready(self, event: Event):

    def on_processing_done(self, event: Event):

Message Passing: Jab specific agent-to-agent communication chahiye, sequence matter kare.
Shared Memory: Jab sab agents ek hi state reference karte hain, read-heavy workloads.
Event-Driven: Jab loosely coupled system chahiye, unknown number of consumers, reactive workflows.

        self.name = name
        self.specialization = specialization

    """Master agent — tasks plan karta hai, delegate karta hai, results collect karta hai"""
    def __init__(self):
        self.workers: List[WorkerAgent] = []
        self.task_queue: List[Task] = []
        self.completed_tasks: List[Task] = []

    def add_worker(self, worker: WorkerAgent):
        self.workers.append(worker)
        print(f"👷 Worker registered: {worker.name} ({worker.specialization})")

    def plan_tasks(self, user_goal: str) -> List[Task]:
        """User goal ko tasks mein break karo"""
        print(f"\n🧠 Planning tasks for: '{user_goal}'")
        tasks = [
            Task("T001", "research", {"query": user_goal}),
            Task("T002", "write",    {"topic": user_goal}),
            Task("T003", "code",     {"feature": user_goal}),
            Task("T004", "review",   {"item": user_goal}),
        ]
        self.task_queue.extend(tasks)
        print(f"  📋 {len(tasks)} tasks planned")
        return tasks

    def delegate(self, task: Task) -> bool:
        """Best worker ko task assign karo"""
        for worker in self.workers:
            if worker.can_handle(task):
                result = worker.execute(task)
                self.completed_tasks.append(task)
                return True

        print(f"  ❌ '{task.task_type}' ke liye koi worker nahi mila!")
        task.status = TaskStatus.FAILED
        return False

    def run(self, user_goal: str):
        """Full workflow run karo"""
        tasks = self.plan_tasks(user_goal)
        print("\n--- Delegation Phase ---")

        for task in tasks:
            self.delegate(task)

        print(f"\n📊 SUMMARY: {len(self.completed_tasks)}/{len(tasks)} tasks complete")

# Consensus Mechanism — Agents vote karte hain decisions pe
# Simple majority voting implementation

from typing import List, Dict, Any
from collections import Counter
import random

class JuryAgent:
    """Ek agent jo vote karta hai"""
    def __init__(self, name: str, bias: float = 0.5):
        self.name = name
        self.bias = bias  # 0=always NO, 1=always YES

    def evaluate(self, proposal: Dict) -> Dict:
        """Proposal evaluate karo aur vote do"""
        # Simplified: bias + some randomness
        score = self.bias + random.uniform(-0.2, 0.2)
        vote = "APPROVE" if score > 0.5 else "REJECT"
        confidence = abs(score - 0.5) * 2  # 0 to 1

        print(f"  🗳️ {self.name}: {vote} (confidence: {confidence:.1%})")
        return {"agent": self.name, "vote": vote, "confidence": confidence}

class ConsensusEngine:
    """Sab votes collect karo aur consensus decide karo"""
    def __init__(self, agents: List[JuryAgent], threshold: float = 0.6):
        self.agents = agents
        self.threshold = threshold  # Majority percentage required

    def reach_consensus(self, proposal: Dict) -> Dict:
        """Proposal pe sab agents se vote lo"""
        print(f"\n🗳️ Consensus Vote: '{proposal.get('name', 'Unknown Proposal')}'")
        print(f"Required majority: {self.threshold:.0%}\n")

        votes = []
        for agent in self.agents:
            vote_result = agent.evaluate(proposal)
            votes.append(vote_result)

        # Count votes
        vote_counts = Counter(v["vote"] for v in votes)
        total = len(votes)
        approve_ratio = vote_counts.get("APPROVE", 0) / total

        consensus_reached = approve_ratio >= self.threshold

        result = {
            "proposal": proposal,
            "votes": votes,
            "approve_count": vote_counts.get("APPROVE", 0),
            "reject_count": vote_counts.get("REJECT", 0),
            "approve_ratio": approve_ratio,
            "consensus_reached": consensus_reached,
            "decision": "APPROVED" if consensus_reached else "REJECTED"
        }

        print(f"\n📊 Results: {result['approve_count']}/{total} approved ({approve_ratio:.0%})")
        print(f"🏆 DECISION: {result['decision']} (threshold: {self.threshold:.0%})")
        return result

🎯 Interview Questions — Communication & Coordination

  1. Q: Message Passing aur Shared Memory mein kya difference hai?
    A: Message passing mein agents directly ek dusre ko messages bhejte hain (point-to-point ya broadcast). Shared memory mein ek common storage hoti hai jahan sab read/write karte hain. Message passing loosely coupled hai, shared memory tightly coupled.
  2. Q: Race condition kya hoti hai aur isko kaise fix karte hain?
    A: Jab do threads/agents ek saath same shared resource access karte hain aur ek dusre ka data overwrite karte hain. Fix: threading.Lock(), mutex, semaphore — ek time pe sirf ek hi access allow karo.
  3. Q: Event-driven architecture ka kya advantage hai?
    A: Loose coupling — publisher aur subscriber ko ek dusre ke baare mein pata nahi hona chahiye. Scalable — naye subscribers add karo bina existing code change kiye. Reactive — events happen hone pe action hota hai, polling nahi.
  4. Q: Consensus algorithms kahan use hote hain real world mein?
    A: Blockchain (Proof of Work/Stake), Distributed Databases (Raft protocol), Kubernetes leader election, Apache ZooKeeper, AI model deployment decisions.

# Hierarchical Architecture Implementation
# Master → Supervisors → Workers — 3 tier system

from typing import List, Dict, Optional
from abc import ABC, abstractmethod

class BaseAgent(ABC):
    def __init__(self, name: str, level: int):
        self.name = name
        self.level = level      # 0=Master, 1=Supervisor, 2=Worker
        self.subordinates = []  # Lower level agents

    def add_subordinate(self, agent):
        self.subordinates.append(agent)

    @abstractmethod
    def execute(self, task: Dict) -> Dict:
        pass

class WorkerAgent(BaseAgent):
    """Level 2 — Actual kaam karta hai"""
    def __init__(self, name: str, skill: str):
        super().__init__(name, level=2)
        self.skill = skill

    def execute(self, task: Dict) -> Dict:
        indent = "    "
        print(f"{indent}⚙️ [{self.name}] (L2): '{task['action']}' execute kar raha hoon")
        result = f"{self.name} completed: {task['action']}"
        print(f"{indent}✅ [{self.name}]: Done!")
        return {"worker": self.name, "result": result}

class SupervisorAgent(BaseAgent):
    """Level 1 — Subtasks workers mein delegate karta hai"""
    def __init__(self, name: str, domain: str):
        super().__init__(name, level=1)
        self.domain = domain

    def execute(self, task: Dict) -> Dict:
        indent = "  "
        print(f"{indent}📋 [{self.name}] (L1): Task split kar raha hoon — '{task.get('description', 'Task')}'")

        # Task ko subtasks mein break karo
        subtasks = task.get("subtasks", [])
        results = []

        for i, subtask in enumerate(subtasks):
            # Round-robin worker assignment
            if not self.subordinates:
                print(f"{indent}⚠️ Koi worker nahi assigned!")
                continue
            worker = self.subordinates[i % len(self.subordinates)]
            result = worker.execute(subtask)
            results.append(result)

        print(f"{indent}✅ [{self.name}]: {len(results)} subtasks complete")
        return {"supervisor": self.name, "domain": self.domain, "results": results}

class MasterAgent(BaseAgent):
    """Level 0 — Overall orchestration, supervisors ko assign karta hai"""
    def __init__(self):
        super().__init__("MasterAgent", level=0)

    def execute(self, task: Dict) -> Dict:
        print(f"👑 [MasterAgent] (L0): High-level task planning")
        print(f"   Goal: {task.get('goal', 'Unknown')}")

        all_results = []
        department_tasks = task.get("department_tasks", [])

        for dept_task in department_tasks:
            # Find matching supervisor by domain
            domain = dept_task.get("domain")
            supervisor = next(
                (s for s in self.subordinates if s.domain == domain), None
            )
            if supervisor:
                print(f"\n  🎯 Delegating '{domain}' to {supervisor.name}")
                result = supervisor.execute(dept_task)
                all_results.append(result)
            else:
                print(f"  ❌ '{domain}' ke liye supervisor nahi mila!")

        print(f"\n✅ [MasterAgent]: All {len(all_results)} departments complete!")
        return {"master": self.name, "department_results": all_results}

# Swarm Intelligence — Simple Particle Swarm Optimization (PSO)
# N agents milke ek mathematical function ka minimum dhundh rahe hain

import random
import math

class Particle:
    """Ek swarm agent — solution space mein move karta hai"""
    def __init__(self, dimensions: int, bounds: tuple):
        low, high = bounds
        # Random starting position
        self.position = [random.uniform(low, high) for _ in range(dimensions)]
        # Random starting velocity
        self.velocity = [random.uniform(-1, 1) for _ in range(dimensions)]
        # Personal best position
        self.best_pos = list(self.position)
        self.best_score = float('inf')

    def update_velocity(self, global_best: list, w=0.7, c1=1.5, c2=1.5):
        """
        Velocity update:
        w  = inertia (apni direction maintain karo)
        c1 = cognitive component (personal best ki taraf)
        c2 = social component (global best ki taraf)
        """
        for i in range(len(self.position)):
            r1, r2 = random.random(), random.random()
            cognitive = c1 * r1 * (self.best_pos[i] - self.position[i])
            social    = c2 * r2 * (global_best[i] - self.position[i])
            self.velocity[i] = w * self.velocity[i] + cognitive + social

    def move(self, bounds: tuple):
        """Position update + bounds check"""
        low, high = bounds
        for i in range(len(self.position)):
            self.position[i] += self.velocity[i]
            # Boundary enforcement
            self.position[i] = max(low, min(high, self.position[i]))

def objective_function(pos: list) -> float:
    """
    Rosenbrock function — ek classic optimization problem
    Minimum at (1, 1) → value = 0
    """
    x, y = pos[0], pos[1]
    return (1 - x)**2 + 100 * (y - x**2)**2

def particle_swarm_optimization(n_particles=30, iterations=50, bounds=(-5, 5)):
    """PSO Algorithm — swarm milke best solution dhundh ta hai"""
    # Initialize swarm
    swarm = [Particle(dimensions=2, bounds=bounds) for _ in range(n_particles)]
    global_best_pos = list(swarm[0].position)
    global_best_score = float('inf')

    print(f"🐝 PSO Shuru! {n_particles} particles, {iterations} iterations")
    print(f"🎯 Goal: f(x,y) = 0 at position (1, 1)\n")

    for iteration in range(iterations):
        # Evaluate each particle
        for particle in swarm:
            score = objective_function(particle.position)

            # Update personal best
            if score < particle.best_score:
                particle.best_score = score
                particle.best_pos = list(particle.position)

            # Update global best
            if score < global_best_score:
                global_best_score = score
                global_best_pos = list(particle.position)

        # Move particles
        for particle in swarm:
            particle.update_velocity(global_best_pos)
            particle.move(bounds)

        # Progress log
        if iteration % 10 == 0:
            x, y = global_best_pos
            print(f"  Iter {iteration:3d}: Best Score={global_best_score:.4f}, "
                  f"Position=({x:.3f}, {y:.3f})")

    print(f"\n🏆 FINAL: Score={global_best_score:.6f}")
    print(f"📍 Best Position: ({global_best_pos[0]:.4f}, {global_best_pos[1]:.4f})")
    print(f"✅ Theoretical optimum: (1.0000, 1.0000)")
    return global_best_pos, global_best_score

result_pos, result_score = particle_swarm_optimization()

    MEDIUM = "medium"  # Log aur proceed

@dataclass
class Action:
    name: str
    description: str

class HITLOrchestrator:
    """
    Human-in-the-Loop Orchestrator
    Risk level ke base pe auto-approve ya human se poochhe
    """
    def __init__(self, auto_approve_below: RiskLevel = RiskLevel.MEDIUM):
        self.auto_approve_below = auto_approve_below
        self.action_log = []
        self.pending_approvals = []

    def _assess_risk(self, action: Action) -> bool:
        """Risk level check — human approval chahiye ya nahi?"""
        risk_order = {RiskLevel.LOW: 0, RiskLevel.MEDIUM: 1, RiskLevel.HIGH: 2}
        auto_level = risk_order[self.auto_approve_below]
        action_level = risk_order[action.risk]
        return action_level > auto_level  # True = human approval needed

    def request_human_approval(self, action: Action) -> bool:
        """
        Real world mein: Slack message, email, dashboard notification
        Yahan simulate karte hain
        """
        print(f"\n  ⚠️ HUMAN APPROVAL NEEDED!")
        print(f"  Action: {action.name}")
        print(f"  Description: {action.description}")
        print(f"  Risk: {action.risk.value.upper()}")
        print(f"  Payload: {action.payload}")
        print(f"  [Simulating: Auto-Approving after review]")

        # In real system: await human input with timeout
        user_response = "APPROVE"  # Simulated
        approved = user_response == "APPROVE"
        return approved

    def execute_action(self, action: Action, executor: Callable) -> Optional[Any]:
        """Action execute karo — risk check ke baad"""
        print(f"\n🔍 Action: '{action.name}' | Risk: {action.risk.value}")

        needs_approval = self._assess_risk(action)

        if needs_approval:
            approved = self.request_human_approval(action)
            if not approved:
                print(f"  ❌ REJECTED by human. Action cancelled.")
                self.action_log.append({"action": action.name, "status": "rejected"})
                return None
            print(f"  ✅ APPROVED by human!")
        else:
            print(f"  ⚡ Auto-approved (low risk)")

        # Execute the action
        result = executor(action.payload)
        self.action_log.append({"action": action.name, "status": "executed", "result": result})
        print(f"  ✅ Executed: {result}")
        return result

📡 5.3 — A2A Protocols (Agent-to-Agent Communication)

Kya hai? A2A (Agent-to-Agent) protocol ek standardized communication format hai jo different AI agents ke beech interoperability ensure karta hai. Jaise HTTP web ke liye hai, A2A multi-agent systems ke liye.

Intuition: Different countries ki apni apni language hoti hai. Lekin United Nations mein sab English mein baat karte hain — ek common protocol. A2A same kaam karta hai AI agents ke liye.

📖 Google ka A2A Protocol (2024-25)

Google ne Agent-to-Agent Protocol propose kiya hai jo standardize karta hai ki different AI agents (different frameworks, different companies) ek dusre se kaise communicate karein. OpenAI ke Agents SDK aur Anthropic ke Model Context Protocol (MCP) bhi is direction mein hain.

A2A Protocol ke Core Components:

🪪

Agent Cards

Har agent ka ek "business card" — name, capabilities, endpoints, authentication requirements. Discovery ke liye use hota hai.

📋

Task Objects

Standardized task format — task ID, instructions, context, expected output format. Koi bhi A2A-compatible agent samajh sake.

🔄

Message Schema

Common message structure — sender, receiver, message type, content, metadata. JSON-based, extensible.

🔐

Auth & Security

JWT tokens, API keys, OAuth — agents ek dusre ko authenticate karte hain before trusting messages.

# A2A Protocol — Simplified Implementation
# Standard format mein agents ek dusre se communicate karte hain

from dataclasses import dataclass, field
from typing import List, Dict, Optional, Any
from datetime import datetime
import uuid

# ===== A2A Data Structures =====

@dataclass
class AgentCard:
    """Agent ka ID card — discovery ke liye"""
    agent_id: str
    name: str
    version: str
    capabilities: List[str]     # "research", "code", "write", etc.
    endpoint: str               # HTTP endpoint ya queue name
    auth_type: str = "api_key"
    max_tasks: int = 10

    def can_handle(self, capability: str) -> bool:
        return capability in self.capabilities

@dataclass
class A2AMessage:
    """Standard A2A message format"""
    message_id: str = field(default_factory=lambda: str(uuid.uuid4())[:8])
    sender_id: str = ""
    receiver_id: str = ""
    message_type: str = "request"  # request|response|notification|error
    protocol_version: str = "1.0"
    timestamp: str = field(default_factory=lambda: datetime.now().isoformat())

    # Content
    task_id: Optional[str] = None
    action: str = ""
    payload: Dict = field(default_factory=dict)
    context: Dict = field(default_factory=dict)

    # Response fields
    status: str = "pending"    # pending|processing|done|error
    result: Optional[Any] = None
    error: Optional[str] = None

@dataclass
class A2ATask:
    """Standardized task jo agents ke beech pass hota hai"""
    task_id: str = field(default_factory=lambda: str(uuid.uuid4())[:8])
    capability_required: str = ""
    instructions: str = ""
    context: Dict = field(default_factory=dict)
    constraints: Dict = field(default_factory=dict)
    expected_output_format: str = "text"  # text|json|markdown

# ===== A2A Registry — Agent Discovery =====

class AgentRegistry:
    """
    Central registry jahan sab agents register karte hain
    Jaise DNS — naam se address dhundho
    """
    def __init__(self):
        self._registry: Dict[str, AgentCard] = {}

    def register(self, card: AgentCard):
        self._registry[card.agent_id] = card
        print(f"📋 Registered: {card.name} | Capabilities: {card.capabilities}")

    def discover(self, capability: str) -> List[AgentCard]:
        """Capability ke basis pe agents dhundho"""
        return [card for card in self._registry.values()
                if card.can_handle(capability)]

    def get_agent(self, agent_id: str) -> Optional[AgentCard]:
        return self._registry.get(agent_id)

# ===== A2A-Compatible Agent =====

class A2AAgent:
    """A2A protocol follow karne wala agent"""
    def __init__(self, card: AgentCard, registry: AgentRegistry):
        self.card = card
        self.registry = registry
        registry.register(card)

    def send_task(self, task: A2ATask) -> A2AMessage:
        """Dusre agent ko task bhejo — discovery ke baad"""
        print(f"\n  📤 [{self.card.name}]: Task bhej raha hoon")
        print(f"  Capability needed: '{task.capability_required}'")

        # Discover capable agents
        capable_agents = self.registry.discover(task.capability_required)
        if not capable_agents:
            return A2AMessage(
                sender_id=self.card.agent_id,
                message_type="error",
                error=f"'{task.capability_required}' ke liye koi agent nahi mila"
            )

        # Best agent select (simplistic: first available)
        target = capable_agents[0]
        print(f"  🎯 Sending to: {target.name} @ {target.endpoint}")

        # Create A2A message
        msg = A2AMessage(
            sender_id=self.card.agent_id,
            receiver_id=target.agent_id,
            message_type="request",
            task_id=task.task_id,
            action=task.capability_required,
            payload={"instructions": task.instructions},
            context=task.context
        )
        return msg

    def handle_request(self, msg: A2AMessage) -> A2AMessage:
        """Incoming request handle karo aur response bhejo"""
        print(f"  📥 [{self.card.name}]: Request receive hua from {msg.sender_id}")

        if not self.card.can_handle(msg.action):
            return A2AMessage(
                sender_id=self.card.agent_id,
                receiver_id=msg.sender_id,
                message_type="error",
                task_id=msg.task_id,
                error=f"I can't handle '{msg.action}'"
            )

        # Process the task
        instructions = msg.payload.get("instructions", "")
        result = f"[{self.card.name}] Completed: {instructions}"

        return A2AMessage(
            sender_id=self.card.agent_id,
            receiver_id=msg.sender_id,
            message_type="response",
            task_id=msg.task_id,
            status="done",
            result=result
        )

from dataclasses import dataclass, field
from typing import List, Dict, Any, Optional
from datetime import datetime

@dataclass
class HandoffPackage:
    """
    Ek agent dusre ko yeh package pass karta hai.
    Poora context + state included hota hai.
    """
    from_agent: str
    to_agent: str
    timestamp: str = field(default_factory=lambda: datetime.now().isoformat())

    # Task context
    original_goal: str = ""
    completed_steps: List[str] = field(default_factory=list)
    remaining_steps: List[str] = field(default_factory=list)

    # Memory & State
    working_memory: Dict[str, Any] = field(default_factory=dict)
    artifacts: Dict[str, str] = field(default_factory=dict)  # Files, outputs

    # Instructions for next agent
    handoff_notes: str = ""
    priority: str = "normal"

    def summary(self):
        return f"""
=== HANDOFF PACKAGE ===
From: {self.from_agent} → To: {self.to_agent}
Goal: {self.original_goal}
Completed: {len(self.completed_steps)} steps
Remaining: {len(self.remaining_steps)} steps
Artifacts: {list(self.artifacts.keys())}
Notes: {self.handoff_notes}"""

class HandoffAgent:
    """Agent jo handoff receive aur send kar sakta hai"""
    def __init__(self, name: str, role: str):
        self.name = name
        self.role = role
        self.current_package: Optional[HandoffPackage] = None

    def receive_handoff(self, package: HandoffPackage):
        """Handoff receive karo — previous context load karo"""
        self.current_package = package
        print(f"\n  📥 [{self.name}] Handoff received from {package.from_agent}")
        print(f"  📖 Handoff notes: {package.handoff_notes}")
        print(f"  📚 Loading {len(package.completed_steps)} completed steps into memory")
        print(f"  🎯 My tasks: {package.remaining_steps[:2]}")

    def do_work(self) -> str:
        """Current task karo"""
        if not self.current_package:
            raise ValueError("No handoff package received!")

        task = self.current_package.remaining_steps.pop(0) if self.current_package.remaining_steps else "generic work"
        result = f"[{self.name}] Completed: '{task}'"
        self.current_package.completed_steps.append(task)
        print(f"  ⚙️ [{self.name}]: {result}")
        return result

    def create_handoff(self, to_agent: str, notes: str) -> HandoffPackage:
        """Apna kaam karke next agent ko handoff create karo"""
        if not self.current_package:
            raise ValueError("No active package to hand off!")

        new_package = HandoffPackage(
            from_agent=self.name,
            to_agent=to_agent,
            original_goal=self.current_package.original_goal,
            completed_steps=list(self.current_package.completed_steps),
            remaining_steps=list(self.current_package.remaining_steps),
            working_memory=dict(self.current_package.working_memory),
            artifacts=dict(self.current_package.artifacts),
            handoff_notes=notes
        )
        print(f"\n  📤 [{self.name}]: Creating handoff to {to_agent}")
        return new_package

💡 Best Practice — Handoff Design

Graceful Handoff ke liye:

  • Always complete current sub-task before handing off — mid-task handoff mein state corrupt ho sakta hai
  • Handoff package mein enough context ho ki next agent bina questions ke continue kar sake
  • Include failure state — agar receiving agent fail ho jaye toh what to do
  • Keep handoff packages serializable — JSON mein convert ho sakein for persistence

🎯 Interview Questions — Advanced MAS Topics

  1. Q: Swarm Intelligence aur Traditional AI mein kya fark hai?
    A: Traditional AI mein ek centralized intelligence hoti hai jo complex rules follow karti hai. Swarm Intelligence mein many simple agents hote hain jo local rules follow karte hain lekin collectively complex, emergent behavior dikhate hain. Intelligence distributed aur decentralized hoti hai. Examples: Ant Colony Optimization, PSO, Boids simulation.
  2. Q: HITL kab necessary hai?
    A: Jab actions irreversible hoin (delete, payment, deployment), jab AI confidence low ho, jab regulatory compliance chahiye, jab edge cases ho jo AI ne nahi dekhe, ya jab high financial/safety stakes hoin. Low-risk repetitive tasks mein HITL overhead add karta hai unnecessarily.
  3. Q: Agent Handoff mein kya data include karna chahiye?
    A: Original goal, completed steps, remaining steps, working memory/context, artifacts (outputs), failure conditions, next agent ke liye instructions, priority level. Basically receiving agent ko bina sender agent se poochhe continue karne mein saksham banana chahiye.
  4. Q: A2A Protocol kyun important hai?
    A: Different companies ke agents (LangChain, AutoGen, CrewAI) naturally ek dusre se communicate nahi kar sakte. A2A standardized interface provide karta hai — jaise HTTP ki wajah se different web servers ek dusre se communicate kar sakte hain, A2A ki wajah se different AI agents ek dusre ko use kar sakte hain.

✅ Best Practices — Multi-Agent System Design

  1. 1

    Start Simple, Scale Later

    Pehle ek agent se start karo. Jab ek agent kafi nahi hota tab dusra add karo. Over-engineering se bachao — "Do you really need 10 agents?" khud se poochho.

  2. 2

    Clear Role Boundaries

    Har agent ka ek clear, specific role hona chahiye. "Research Agent" sirf research kare — writing nahi. Role overlap confusion aur conflicts create karta hai.

  3. 3

    Observability se Start Karo

    Logging, tracing, aur monitoring pehle day se add karo. Production mein ek agent kya kar raha hai yeh bina logs ke impossible hai track karna.

  4. 4

    Idempotent Tasks Design Karo

    Tasks aise design karo ki agar ek agent fail ho aur dusra usi task ko retry kare, toh duplicate results na aayein. "Write file if not exists" vs "Write file" — pehla idempotent hai.

  5. 5

    Timeout aur Fallback

    Har agent call pe timeout set karo. Agent respond nahi kare toh kya karna hai — fallback logic likho. Infinite wait production mein disaster hai.

  6. 6

    Test Each Agent Independently

    Pehle har agent ko isolated test karo, phir integration test karo. Multi-agent bugs dhundna mushkil hota hai — unit testing se early catch hoga.

  7. 7

    Context Window Budget Manage Karo

    Agents ke beech poora context pass karte rehne se context window quickly fill ho jaata hai. Sirf relevant information pass karo — handoff packages concise rakhte hain.

  8. 8

    Human Checkpoints Add Karo

    Bade, complex MAS mein har milestone pe ek human checkpoint add karo. Fully autonomous system mein ek choti galti bhi downstream mein badi problem ban sakti hai.

⚠️ Common Mistakes — Multi-Agent Systems

❌ Mistakes to Avoid

1. Agent Hell: Zyada agents banana sirf kyunki "agents cool lagte hain." Har ek agent overhead add karta hai — API calls, latency, cost.

2. Context Loss: Handoff mein important context miss karna — next agent "blind" start karta hai.

3. Circular Dependencies: Agent A → Agent B → Agent C → Agent A. Infinite loop ban jaata hai.

4. No Error Handling: Ek agent ka failure poora pipeline rok deta hai. Graceful degradation design karo.

5. Monolithic Orchestrator: Orchestrator mein saari business logic mat daalo. Agents ko smarter banao.

6. Race Conditions: Shared memory mein locking implement nahi karna — data corruption guaranteed.
