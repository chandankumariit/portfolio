📘 Phase 1 — AI, Agents aur Agentic Systems

🧠 What is AI / GenAI? beginner first

Artificial Intelligence (AI) — machine ko insani tareeke se sochna sikhana. Generative AI (GenAI) naya content create karta hai: text, image, code. Jaise ChatGPT, Gemini.

⚡ Simple Hindi: AI = machine mein dimaag daalna. GenAI = naye ideas / baat / picture banana.

🤖 What is an AI Agent?

AI Agent = LLM + Tools + Memory + Planning — Bas ek chatbot nahi, yeh khud decision leta hai, tools use karta hai, aur ek goal complete karne ke liye actions karta hai.

┌─────────────┐ ┌─────────────┐ │ GOAL │ ──→ │ REASONING │ │ "Book a ticket" │ │ (क्या करना है?) │ └─────────────┘ └──────┬──────┘ │ ▼ ┌─────────────┐ ┌─────────────┐ │ ACTION │ ←── │ PLANNING │ │ (search flight) │ (step by step) │ └─────────────┘ └─────────────┘

Chatbot vs Agent: Chatbot sirf baat kare; agent kaam kare — database query kare, email bheje, code execute kare.

✅ Single‑Agent System

Ek agent sab kuch kare: planning + tools + memory. Simple tasks ke liye best.
Example: Personal assistant (Alexa + actions).

👥 Multi‑Agent System (MAS)

Multiple agents, har agent specialist. Ek planner, ek coder, ek reviewer. Collaborate karte hain complex problem solve karne ke liye.

⚙️ Phase 2 — LLMs: Agent ka dimaag

Transformer + Attention intuition

Transformer ek architecture hai jisse LLM (GPT etc.) bane. Attention mechanism har word ko doosre words ko "dekhne" ki power deta hai — context samajhne ke liye.

💡 "The cat sat on the mat because it was soft." → 'it' mat ki baat kar raha hai. Attention se model connect karta hai.
// PyTorch style pseudo: Attention(Q,K,V) = softmax(QK^T/√d) * V // Q = Query (main token kya dhundh raha) // K = Key (value ka label) // V = Value (actual content)

🔧 Function calling / Tool calling — Agent ki superpower

LLM sirf text nahi generate karta, aaj ke models function signatures samajhte hain. Jab agent ko lagta hai "bahar jaana padega" (weather API, calculator), toh woh function call karta hai.

User: "Delhi ka aaj ka temperature kya hai?" │ ▼ LLM → decides: tool = get_weather(location="Delhi") │ ▼ Tool execution → returns "32°C" │ ▼ LLM: "Delhi mein aaj 32°C hai."

Structured output — JSON ya pydantic object automatically banwana. Agents ke liye zaroori.

🎼 Phase 4 — Orchestration: Multi-Agent ka conductor

Orchestration = multiple agents, tools, memory kaise coordinate ho? Ye engine tasks ko route karta hai, state maintain karta hai, failure handle karta hai.

🎯 Importance: bina orchestration ke agents randomness me kaam karenge. Orchestrator DAG (directed acyclic graph) banata hai — step1, step2 parallel, etc.
┌─────────────────────────────────────────┐ │ ORCHESTRATOR ENGINE │ │ (LangGraph / Custom State Machine) │ └─────────────────────────────────────────┘ │ │ │ ┌───────┼─────────┼─────────┼───────┐ ▼ ▼ ▼ ▼ ▼ Agent1 Agent2 Tool DB Memory Router

Workflow types:

  • ✅ Sequential (A→B→C)
  • ✅ Parallel (A and B simultaneously)
  • ✅ Conditional (if-else routing)
  • ✅ Loop/Retry (reflection cycles)

🚀 FINAL: Production‑Ready Multi‑Agent Orchestrator Code (Python + LangGraph)

# ⚡ Complete Multi-Agent System with Tools + Memory
# pip install langgraph langchain-openai chromadb

import operator
from typing import Annotated, List, TypedDict
from langgraph.graph import StateGraph, END
from langchain_openai import ChatOpenAI
from langchain.tools import tool
from langchain.memory import ConversationBufferMemory
from langchain_core.messages import HumanMessage, AIMessage

# ---------- TOOLS ----------
@tool
def search_web(query: str) -> str:
    """Real world web search mock"""
    return f"Search results for '{query}': Latest news about AI agents."

@tool
def calculate(expression: str) -> str:
    """Calculator tool"""
    return str(eval(expression))

tools = [search_web, calculate]

# ---------- AGENT STATE ----------
class AgentState(TypedDict):
    messages: Annotated[List, operator.add]
    next_step: str
    final_answer: str

llm = ChatOpenAI(model="gpt-4o", temperature=0).bind_tools(tools)

# ---------- NODES ----------
def planner_node(state: AgentState):
    """Planning agent: break down goal"""
    prompt = "You are a planner. Break down user request into steps."
    response = llm.invoke([HumanMessage(content=prompt + "\n" + state["messages"][-1].content)])
    return {"messages": [response], "next_step": "executor"}

def executor_node(state: AgentState):
    """Executor: use tools if needed"""
    last_msg = state["messages"][-1].content
    response = llm.invoke([HumanMessage(content=f"User goal: {last_msg}. Use tools if needed.")])
    # tool calling automatically handled by LLM bind_tools
    return {"messages": [response], "next_step": "aggregator"}

def aggregator_node(state: AgentState):
    """Combine all outputs"""
    final = llm.invoke([HumanMessage(content=f"Summarize final answer: {state['messages']}")])
    return {"final_answer": final.content, "next_step": END}

# ---------- GRAPH ----------
builder = StateGraph(AgentState)
builder.add_node("planner", planner_node)
builder.add_node("executor", executor_node)
builder.add_node("aggregator", aggregator_node)

builder.set_entry_point("planner")
builder.add_edge("planner", "executor")
builder.add_edge("executor", "aggregator")
builder.add_edge("aggregator", END)

graph = builder.compile()

# ---------- RUN ----------
if __name__ == "__main__":
    initial_state = {"messages": [HumanMessage(content="What is 125 * 9 and also search latest AI news?")], "next_step": "", "final_answer": ""}
    result = graph.invoke(initial_state)
    print("🤖 Agent Final Answer:\n", result["final_answer"])
